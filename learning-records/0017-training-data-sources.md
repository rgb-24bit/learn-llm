# 0017: 训练数据的来源与优化

**日期**: 2026-07-18
**来源**: 阶段5 教学内容——训练数据篇

## 关键要点

### 数据来源
- LLM 训练数据 80%+ 来自 **Common Crawl**（互联网快照），其余来自 Wikipedia/书籍/论文/代码/社交媒体
- 每个来源给模型带来不同的能力：Wikipedia=知识、书籍=长篇推理、代码=逻辑思维
- 光靠 Wikipedia 远远不够——整个英文 Wikipedia 才约 40 亿词，不到 Llama 3 训练数据的 1/3000

### 清洗流水线（5 步）
1. **HTML 提取** — 去标签、广告、导航栏
2. **语言识别** — FastText 判断文档语言
3. **启发式过滤** — 长度/符号比例/重复率等规则（FineWeb 从 50+ 规则中只保留了 7 条最有效的）
4. **去重** — 精确去重 + MinHash 模糊去重（全局去重 > 局部去重）
5. **模型质量过滤** — 用 Llama 3 70B 标注 + 小分类器批量打分（FineWeb-Edu 的方法）

### 模型质量过滤（FineWeb-Edu）
- 用 Llama 3 70B 给 50 万篇文档标注 1-5 分（教育质量）
- 训练一个 5 层 fastText 分类器 → 扫整个 FineWeb（15T token）
- 保留分数 ≥ 3 的文档（约 8.7%）
- 结果：MMLU 提升 5-10 个百分点
- 代价：筛掉 90% 数据，数据量不够大模型长期训练

### 数据混合与 Chinchilla 法则
- 不同来源按比例混合（代码不能少、书籍比例要多、Wikipedia 量小值钱）
- Chinchilla 的 20 token:1 参数的前提是数据质量够高
- 高质量数据可以"节约"训练量

### 数据污染与去污染
- 测试集混入训练数据是普遍问题（因为测试集也公开在网上）
- 精确匹配可检测，变体污染（改写版）几乎无法检测
- 动态基准（LiveBench）是终极解决方案

### SFT 数据
- 质量 > 数量：1,000 条精心挑选的 SFT 数据可以比 50,000 条未筛选的好
- InstructGPT 只用 13K 条人工标注
