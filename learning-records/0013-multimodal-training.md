# 0013: 多模态训练——图文对齐与多模态模型架构

**日期**: 2026-07-16
**来源**: CLIP (arXiv:2103.00020) + LLaVA (arXiv:2304.08485) + Qwen2-VL (arXiv:2409.12191)

## 关键要点

### CLIP 图文对齐（核心机制）
- 图片和文字本身是两套"语言"，要让模型理解图文关系，需要**对比学习**
- 训练方式：拿 N 对图文对 → 图片编码器和文本编码器各自输出 N 个向量 → 算 N×N 相似度矩阵 → 拉近对角线（配对项）、推远非对角线
- 结果：猫的照片和"猫"这个文字，在 embedding 空间里距离很近

### 多模态 LLM 的"三件套"
1. **视觉编码器**（ViT）— 把图片变成视觉 token
2. **连接器**（MLP / Cross-Attention / Q-Former）— 把视觉 token 翻译到 LLM 的空间
3. **LLM 主干**（LLaMA / Qwen 等）— 统一处理视觉 token + 文本 token

### 多模态训练的三个阶段
- **阶段 A（对齐预训练）**：只用图文对数据，只训练连接器（投影层），冻结 ViT + LLM
- **阶段 B（多模态 SFT）**：用图片问答数据，训练连接器 + LLM（LoRA 或全量）
- **阶段 C（偏好对齐）**：跟纯文本的 DPO/RLHF 类似，但扩展到了图像场景

### 与纯文本 LLM 训练的关系
- CLIP 预训练 ≈ 纯文本的 Pretrain（建立图文关联的基础能力）
- 多模态 SFT ≈ 纯文本的 SFT（教会模型看图回答问题）
- 关键差异：纯文本 LLM 的 pretrain 直接学到知识，多模态模型的"看图能力"来自 CLIP 的预训练

### 数值对比
| 维度 | 纯文本 LLM 训练 | 多模态训练（补充） |
|:-----|:---------------|:-----------------|
| 预训练数据量 | TB 级文本 | 4 亿图文对 (CLIP) |
| 新增参数 | — | 20M 投影层 |
| 训练时长 | 数周~数月 | 1-8 天（单卡/小集群） |
| 训练阶段 | Pretrain → SFT → DPO | CLIP → 对齐 → 多模态SFT → DPO |
