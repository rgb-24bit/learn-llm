# 04: NTK-aware 与 YaRN——上下文窗口扩展

**日期**: 2026-07-09
**来源**: minimind `model_minind.py` rope_scaling + 论文

## 关键要点

### 为什么需要扩展
训练时模型只见过固定长度（如 2048），推理时想处理更长的输入（如 32k）。RoPE 的频率训练窗口之外还有数学定义，但需要调整才能用好。

### 三种方法
1. **位置插值（PI）**：新位置缩回原范围。问题：高频维度被压缩，近处分不清
2. **NTK-aware**：利用不同维度频率不同的特性——高频不缩（保留近处分辨力），低频缩（反正它本来也看不清近处）
3. **YaRN**：NTK 基础上 + Attention logits 缩放补偿

### 核心直觉
高频 = 每秒转 1 圈的显微镜，低频 = 每 10000 秒转 1 圈的望远镜。压缩时保留显微镜的精度只缩望远镜，效果最好。
