# 03: RoPE 旋转位置编码

**日期**: 2026-07-09
**来源**: minimind `apply_rotary_pos_emb` + LLMs-from-scratch ch03

## 关键要点

### RoPE 做了什么
在 Attention 计算之前，对 Q 和 K 按位置做二维旋转。每个 token 的位置不同，旋转角度不同。

### 配对方式
把 Q/K 向量分成前后两半配对：(d₀,d₄), (d₁,d₅), (d₂,d₆), ... 每对在各自的二维平面上旋转。

### 高频 vs 低频
- 编号小的配对旋转快（高频）→ 能精细区分邻近位置
- 编号大的配对旋转慢（低频）→ 只能区分远距离
- 48 个不同频率合起来从近到远全覆盖

### 为什么旋转不改变语义
**旋转不改变向量长度**，只改变方向。方向变 = 位置指纹加上了，长度不变 = 语义强度没丢。

### 相对位置感知
旋转后的 Q 和 K 做点积，结果天然依赖 cos(位置差)——距离越远关注度越低。模型不需要额外学习就能感知相对位置。

## 绝对位置编码（GPT-2 风格）vs RoPE
| | 绝对位置 | RoPE |
|:--|:---------|:-----|
| 在哪加 | 加到 token embedding 上 | 旋转 Q/K |
| 位置 2049 | 没学过→不能用 | 可扩展到 32k+ |
| 相对位置 | 要靠学 | 天然有 |
