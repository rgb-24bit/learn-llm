# 📈 Scaling Law——规模定律

> 第三阶段 — Pretrain 训练扩展
> 2026-07-13

## 核心发现（OpenAI 2020）

模型性能（PPL）与规模在对数坐标下呈**直线关系**（幂律）。

```
PPL ∝ N^(-0.076)   ← 参数量翻倍，PPL 降约 5%
PPL ∝ D^(-0.095)   ← 数据量翻倍，PPL 降约 6%
PPL ∝ C^(-0.050)   ← 计算量翻倍，PPL 降约 3.5%
```

## 三根杠杆

| 杠杆 | 单位 | 效果 |
|:----|:----|:----|
| N（参数量） | 权重个数 | 模型容量 |
| D（数据量） | token 数 | 泛化能力 |
| C（计算量） | FLOPs | 总预算 = N × D × 训练步数 |

## Chinchilla 法则（DeepMind 2022）

**参数和数据必须等比缩放，最优比例 ≈ 1:20。**

- GPT-3：175B 参数 × 300B token → 每个参数只看到 1.7 个 token → **欠数据**
- Chinchilla：70B 参数 × 1.4T token → 每个参数看到 20 个 token → **最优**
- 同样计算预算下，Chinchilla 效果优于 GPT-3

## 边际递减

```
1M → 10M：PPL 降幅大    ← 容易
10M → 100M：还行        ← 可以
100M → 1B：开始困难     ← 数据不够了
1B → 100B：很小提升     ← 天花板
100B → 1T：几乎不动     ← 极限
```

## 小模型的行为（MobileLLM）

固定参数下，**深度 > 宽度**。

- d_model < 512：表示瓶颈，加层数无效
- d_model 512~768：窄深模型效果最好
- d_model > 1536：加层数优于加宽度
