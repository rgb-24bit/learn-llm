> 目标：每一步都知道"输入什么 → 算什么 → 输出什么"

## 第一部分：一点就够的基础数学

### ① "乘完再加"就能读懂 Attention

```
两个数组 [a, b, c] 和 [d, e, f]
"乘完再加" = a×d + b×e + c×f

例子：
[3, 1, 4] · [2, 7, 1] = 3×2 + 1×7 + 4×1 = 6 + 7 + 4 = 17
```

越大 = 越像（方向越接近）
越小 = 越不像（方向越远）
0 = 不相关（垂直）

### ② "表格×数组" = 做好多遍"乘完再加"

```
一个 2行3列 的表格，乘一个 3个数 的数组：

W = | 0.5  0.2  0.3 |      x = [1, 2, 3]
    | 0.1  0.4  0.6 |

第一遍：第一行 × x = 0.5×1 + 0.2×2 + 0.3×3 = 0.5+0.4+0.9 = 1.8
第二遍：第二行 × x = 0.1×1 + 0.4×2 + 0.6×3 = 0.1+0.8+1.8 = 2.7

结果 = [1.8, 2.7]
```

就是这么简单，一遍一遍做"乘完再加"。

### ③ 三角函数只用两个数

```
角度     cos     sin
 0°      1       0
45°     0.71    0.71
90°      0       1
180°    -1       0
270°     0      -1
360°     1       0
```

cos = 横坐标，sin = 纵坐标，在圆上转圈圈。

### ④ 二维旋转——拿来用就行

```
把点 (x, y) 转 θ 度：
新 x = x×cosθ - y×sinθ
新 y = x×sinθ + y×cosθ

例：点 (1, 0) 转 90°
新 x = 1×0 - 0×1 = 0
新 y = 1×1 + 0×1 = 1
→ (1, 0) → (0, 1) ✅ 指向上面了

！！旋转不改变长度！！
1²+0²=1，0²+1²=1 ✅ 一样长
```

## 第二部分：QKV 的计算过程

### 准备工作

```
句子 "我 爱 你"

第一步：把每个词变成 4 个数字（简化，真实是 768 个）
"我" → [0.5, 0.1, 0.3, 0.8]
"爱" → [0.2, 0.7, 0.1, 0.4]
"你" → [0.6, 0.3, 0.5, 0.2]

第二步：准备三个 4行4列 的表格
W_Q = 随机数字（训练中逐渐学会"怎么提问"）
W_K = 随机数字（训练中逐渐学会"怎么展示自己"）
W_V = 随机数字（训练中逐渐学会"提供什么信息"）
```

### Step 1：每个词×W_Q → Q（我在找什么）

```
"我" × W_Q：
W_Q = | 0.1  0.2  0.3  0.4 |     "我" = [0.5, 0.1, 0.3, 0.8]
      | 0.2  0.5  0.1  0.3 |
      | 0.3  0.4  0.2  0.1 |
      | 0.4  0.3  0.5  0.2 |

第一行："乘完再加"
  0.1×0.5 + 0.2×0.1 + 0.3×0.3 + 0.4×0.8
  = 0.05 + 0.02 + 0.09 + 0.32 = 0.48

第二行：
  0.2×0.5 + 0.5×0.1 + 0.1×0.3 + 0.3×0.8
  = 0.10 + 0.05 + 0.03 + 0.24 = 0.42

第三行：... = 0.32
第四行：... = 0.51

→ Q_我 = [0.48, 0.42, 0.32, 0.51]
   ← "我"在找什么：这 4 个数字编码了"我在找动词 / 我在找对象..."
```

同理算 K（"我"有什么可被找到的）和 V（"我"能提供什么）。

### Step 2：所有词的 Q 和 K

```
Q_我 = [0.48, 0.42, 0.32, 0.51]  K_我 = [0.55, 0.38, 0.29, 0.46]  V_我 = [0.33, 0.52, 0.41, 0.37]
Q_爱 = [0.27, 0.41, 0.29, 0.33]  K_爱 = [0.41, 0.32, 0.28, 0.43]  V_爱 = [0.45, 0.28, 0.51, 0.39]
Q_你 = [0.35, 0.44, 0.39, 0.38]  K_你 = [0.42, 0.40, 0.33, 0.41]  V_你 = [0.36, 0.43, 0.38, 0.42]
```

### Step 3：Q 和 K "乘完再加"→ 关注度分数

```
"我"关注"爱"：
Q_我·K_爱 = 0.48×0.41 + 0.42×0.32 + 0.32×0.28 + 0.51×0.43
          = 0.197 + 0.134 + 0.090 + 0.219
          = 0.640  ← 较高，"我"觉得"爱"有关系

"我"关注"你"：
0.48×0.42 + 0.42×0.40 + 0.32×0.33 + 0.51×0.41 = 0.685

"我"关注自己：
0.48×0.55 + 0.42×0.38 + 0.32×0.29 + 0.51×0.46 = 0.752 ← 最高

→ 分数矩阵（3行3列）：
           我      爱      你
   我     0.752   0.640   0.685
   爱     0.582   0.541   0.601
   你     0.663   0.598   0.617
```

### Step 4：除√4（把数字变小，不让某一个太大）

```
√4 = 2，每个数 ÷2：

           我      爱      你
   我     0.376   0.320   0.343
   爱     0.291   0.271   0.301
   你     0.332   0.299   0.309
```

### Step 5：softmax → 每行加起来 = 1

```
"我"行 [0.376, 0.320, 0.343]：

第1步：算 e^x（e≈2.718）
  e^0.376=1.456, e^0.320=1.377, e^0.343=1.409

第2步：求和 1.456+1.377+1.409 = 4.242

第3步：每个÷和
  1.456/4.242=0.343, 1.377/4.242=0.325, 1.409/4.242=0.332

→ 关注度："我"占34.3%，"爱"占32.5%，"你"占33.2%
   "我"和大家都有关联，所以差不多平均分配
```

### Step 6：用关注度对 V 做"加权平均"→ 新向量

```
"我"的注意力：
  34.3% 看自己：V_我 = [0.33, 0.52, 0.41, 0.37]
  32.5% 看"爱"：V_爱 = [0.45, 0.28, 0.51, 0.39]
  33.2% 看"你"：V_你 = [0.36, 0.43, 0.38, 0.42]

新"我" = 0.343×V_我 + 0.325×V_爱 + 0.332×V_你

位置0：0.343×0.33 + 0.325×0.45 + 0.332×0.36 = 0.113+0.146+0.120 = 0.379
位置1：0.343×0.52 + 0.325×0.28 + 0.332×0.43 = 0.178+0.091+0.143 = 0.412
位置2：0.343×0.41 + 0.325×0.51 + 0.332×0.38 = 0.141+0.166+0.126 = 0.433
位置3：0.343×0.37 + 0.325×0.39 + 0.332×0.42 = 0.127+0.127+0.139 = 0.393

→ 新"我" = [0.379, 0.412, 0.433, 0.393]
   ← "我"融合了"我"+"爱"+"你"的信息，更新了自己
```

**Attention 的本质**：每个词看完整个句子后，带着上下文信息更新自己。

## 第三部分：RoPE 的实际计算

### 维度配对方式

```
一个 8 维的向量（简化，真实 96 维）
[d₀, d₁, d₂, d₃, d₄, d₅, d₆, d₇]

RoPE 把向量分成前后两半配对：
前半：[d₀, d₁, d₂, d₃]
后半：[d₄, d₅, d₆, d₇]

配对：
对 0：(d₀, d₄)  共用同一个旋转速度
对 1：(d₁, d₅)  共用同一个旋转速度
对 2：(d₂, d₆)  共用同一个旋转速度
对 3：(d₃, d₇)  共用同一个旋转速度

一共 4 对（真实是 48 对），每对速度不同
```

### 每对的速度怎么算的？

只看具体数字怎么来的（不解释公式）：

```
dim=96，共 48 对（编号 0 到 47）

对 0：  1/10000^0         = 1/1       = 1.0     ← 最快
对 1：  1/10000^0.021     ≈ 1/1.1     ≈ 0.91
对 2：  1/10000^0.042     ≈ 1/1.2     ≈ 0.82
对 3：  1/10000^0.063     ≈ 1/1.3     ≈ 0.75
对 4：  1/10000^0.083     ≈ 1/1.5     ≈ 0.68
对 5：  1/10000^0.104     ≈ 1/1.6     ≈ 0.62
...                           ↓ 越往后越小 ↓
对 12： 1/10000^0.25      = 1/3.2     ≈ 0.32   ← 中速
...                           ↓ 越往后越小 ↓
对 24： 1/10000^0.5       = 1/100     ≈ 0.01   ← 很慢
...
对 47： 1/10000^0.979     ≈ 1/9500    ≈ 0.0001 ← 极慢
```

**为什么 0 是高频、47 是低频？** 因为编号越大，幂次（指数）就越大，10000 的指数稍微变大，结果就剧烈缩小。1/10000^0=1 和 1/10000^1=1/10000，差了 10000 倍！所以 48 步就覆盖了从 1 到 0.0001 的宽范围。

### 具体算一次旋转

```
位置 p=3（第 4 个词）

查位置 3 的 cos/sin 表（预计算好的，直接拿来用）：
对 0（速度 1.0）：3×1.0=3 rad≈172°, cos=-0.99, sin=0.14
对 1（速度 0.56）：3×0.56=1.68 rad≈96°, cos=-0.10, sin=0.99
对 2（速度 0.32）：3×0.32=0.96 rad≈55°, cos=0.57, sin=0.82
对 3（速度 0.18）：3×0.18=0.54 rad≈31°, cos=0.86, sin=0.52
```

只看对 0（d₀, d₄）怎么转：

```
Q = [d₀=0.5,  d₁=0.3,  d₂=0.1,  d₃=0.8,  d₄=0.2,  d₅=0.7,  d₆=0.4,  d₇=0.6]

对 0 要旋转 (d₀, d₄)：
原来 d₀=0.5, d₄=0.2
位置 3 在 cos/sin 表上：cos=-0.99, sin=0.14

套用二维旋转公式（只问怎么算，不问为什么）：
新 d₀ = 0.5×(-0.99) - 0.2×0.14 = -0.495 - 0.028 = -0.523
新 d₄ = 0.2×(-0.99) + 0.5×0.14 = -0.198 + 0.070 = -0.128

点 (0.5, 0.2) → 转 172° → (-0.523, -0.128)

验证长度没变：
√(0.5²+0.2²) = √0.29 = 0.539
√((-0.523)² + (-0.128)²) = √(0.274+0.016) = √0.290 = 0.539 ✅
```

### 高频和低频的实际效果差异（用数字看）

**高频（对 0，速度 1.0）**：
```
位置差=1：cos(1×1.0)=cos(1)=0.54    ← 降了快一半！
位置差=2：cos(2×1.0)=cos(2)=-0.42   ← 变成负数了
位置差=3：cos(3×1.0)=cos(3)=-0.99   ← 几乎反向

→ 变化剧烈：能精细区分"位置差1"和"位置差2"
```

**低频（对 47，速度 0.0001）**：
```
位置差=1：cos(1×0.0001)=cos(0.0001)≈1.0   ← 几乎没变！
位置差=100：cos(100×0.0001)=cos(0.01)≈1.0 ← 还是几乎没变
位置差=10000：cos(10000×0.0001)=cos(1)=0.54 ← 终于降了一点

→ 变化平缓：只能区分"很近"和"很远"
```

**合在一起**：
- 高频负责"紧挨着的两个位置我分得清"
- 低频负责"隔了老远我还知道距离感"
- 48 个频率合起来，从近到远全覆盖

## 第四部分：训练时三个表格怎么学会的

```
刚开始：
W_Q, W_K, W_V 全是随机数
→ Q"瞎问"，K"乱答"，V"瞎提供"

训练一步（看一个句子 "我 爱 你"）：

模型预测下一个词 → 猜错了 → 算误差
误差往回传 → 调整 W_Q, W_K, W_V 里的每个数字

训练了一亿步后：
W_Q 学会了"动词位置的 Q 应该问：我的主语是谁？"
W_K 学会了"名词位置的 K 应该回答：我是主语候选"
W_V 学会了"名词位置被关注时提供：我是第几个人称"
```

**W_Q, W_K, W_V 不是设计出来的，是训练出来的。** 它们学会什么完全取决于训练数据 (´｡• ᵕ •｡`)✧
