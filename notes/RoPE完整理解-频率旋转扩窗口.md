## 一、RoPE 的核心机制

### 1.1 频率公式

```python
freqs[k] = 1.0 / (rope_base ** (2*k / dim))
```

| 参数 | minimind 值 | 含义 |
|:----|:----------:|:----|
| dim（head_dim） | 96 | 每个注意力头的维度 |
| k | 0~47 | 频率对编号，数量 = dim/2 |
| rope_base | 10000 | 控制整体频率快慢 |

### 1.2 配对方式

```
Q 向量（96维）：[q₀, q₁, ..., q₄₇, q₄₈, q₄₉, ..., q₉₅]
                  前半 48 个      ─┬─    后半 48 个
                                   │
配对：                             │
  (q₀, q₄₈)  ← k=0, 速度 1.0（最快）
  (q₁, q₄₉)  ← k=1, 速度 0.91
  ...
  (q₄₇, q₉₅)  ← k=47, 速度 0.0001（最慢）
```

**不是相邻配对，是前半跟后半配对。**

### 1.3 旋转计算

位置 p 时，对每对 (x, y) 做二维旋转：

```
旋转角度 θ_k(p) = p × freq_k

新 x = x × cos(θ) - y × sin(θ)
新 y = y × cos(θ) + x × sin(θ)
```

**cos/sin 是预计算好的固定值（预计算好查表）**，不是可训练参数。

## 二、高频 vs 低频

### 2.1 划分方式

没有一刀切的边界，是连续光谱。

| 范围 | 训练窗口 2048 内转了多少 | 负责什么 |
|:----|:---------------------:|:--------|
| k=0~15（高频） | 几十~几百圈 | 区分邻近位置（差 1、差 2、差 3） |
| k=16~36（中频） | 几圈 | 区分中距离（差 10、差 100） |
| k=37~47（低频） | 不到 1 圈 | 区分远距离（差 1000+） |

### 2.2 频率的本质

```
频率高 = cos(位置差 × freq) 随位置差变化快
频率低 = cos(位置差 × freq) 随位置差变化慢

k=0：d+1 → cos 跳 ~1 rad    → 能分辨差 1 vs 差 2
k=24：d+100 → cos 跳 ~1 rad → 能分辨差 100 vs 差 200
k=47：d+10000 → cos 跳 ~1 rad → 能分辨差 10000 vs 差 20000
```

**48 个频率合一 = 一个位置差产生 48 维的指纹向量。**
在训练窗口内，每个位置差的指纹都是唯一的。

### 2.3 转盘类比

```
48 个转盘，从里到外：
  最里圈（k=0）：转得最快，6 步回到起点
  中间圈（k=24）：转得中等，600 步回到起点
  最外圈（k=47）：转得最慢，60000 步回到起点

每个位置 p = 所有转盘同步转了一步后的大合照。
不同位置的合照都不同（48 个转盘同时回到原位几乎不可能）。
```

## 三、RoPE 的完整工作流程

### 3.1 前向传播

```
token_embedding（在位置 p）
    ↓
× W_Q → 原始 Q      × W_K → 原始 K
    ↓                      ↓
旋转（根据位置 p）        旋转（根据位置 p）
    ↓                      ↓
Q_rot                   K_rot  ← 点积之前旋转好
    │                      │
    └──────── Q_rot @ K_rot^T ────────→ 点积结果携带位置差信息
```

**Q 和 K 都旋转，角度完全一样。V 不旋转。**

### 3.2 为什么 Q 和 K 都要转

```
Q_i · K_j = 原 Q_i·K_j × cos((i-j) × freq_k) + ...
                                     ↑
                            完全由位置差决定 ✅

如果只转 Q、K 没转：点积结果没有规律 → 模型学不会 ❌
```

### 3.3 不同注意力头的关系

```
所有 8 个头共用同一套旋转频率表（同一张 cos/sin 查表）。
但每个头有自己的 W_Q 和 W_K。

旋转的「速度盘子」8 个头共用，「被旋转的向量」各头不同。
```

### 3.4 反向传播

**cos/sin 是固定常数，不参与梯度更新。**

```
前向：q_rot = q_raw × cos + rotate_half(q_raw) × sin
                              ↑                   ↑
                          都是常数（不参与 BP）

反向：梯度穿过旋转操作时，cos/sin 只是矩阵乘法系数
      最终回到 W_Q 和 W_K，正常更新

W_Q/W_K 从数据中学到「旋转不变性」——
不管 Q/K 被旋转了多少度，学好语义关系才是关键。
```

## 四、KV Cache 与旋转

```python
xq, xk = self.q_proj(x), self.k_proj(x)
xq, xk = apply_rotary_pos_emb(xq, xk, cos, sin)  # 先旋转

if past_key_value is not None:
    xk = torch.cat([past_key_value[0], xk], dim=1)  # 拼接时已经是旋转后的 K

past_kv = (xk, xv) if use_cache else None  # 缓存旋转后的 K
```

**KV Cache 缓存的是旋转后的 K。** 每步只需旋转新 token 的 K，历史 K 直接复用。

## 五、上下文窗口扩展

### 5.1 为什么 RoPE 能扩

RoPE 不是查表，是按公式算角度。公式里的 rope_base（10000）可以调。

### 5.2 三种扩窗方法

**PI（Position Interpolation）——最土**
```
把 4096 个位置压缩进 2048 个位置的旋转角度
代价：邻近区分度降低
```

**NTK-aware——聪明折中**
```
高频对（k 小）：几乎不动 ← 保留邻近精度
低频对（k 大）：大幅度调慢 ← 让新位置角度落回已见范围
```

**YaRN——最好**
```
在 NTK 基础上：
1. 频率调慢在中间范围平滑过渡
2. 调整 attention softmax 温度
```

### 5.3 训练窗口 vs 推理窗口

| 模型 | 训练窗口 | 推理窗口 | 方法 |
|:----|:-------:|:-------:|:----|
| LLaMA 3.1 | 8k | 128k | YaRN |
| Mistral | 8k | 32k | NTK-aware |
| DeepSeek V3 | 4k | 128k | YaRN |
| Qwen 2.5 | 32k | 128k~1M | YaRN + 长文本微调 |

**200k 窗口绝大多数是 8k 训练出来的。**

### 5.4 cos 的周期性不会造成混淆

```
单一 k 对 cos 会周期重复（周期 2π）。
但 48 个 k 一起的指纹不重复——在训练窗口内是唯一的。

高频（k 小）负责近处，还没到周期就已区分完毕。
低频（k 大）负责远处，每个位置差都是唯一角度。
48 合一全局唯一。
```

### 5.5 窗口扩展的极限

```
高频对（k 小）：转得飞快，cos 值大量重复
  → 本来也不靠它们分远处，不影响

低频对（k 大）：200k 位置才转 3.2 圈
  → 角度还没重复，仍能分远处

极限由低频对的维度数（dim/2）决定。
这也为什么 DeepSeek 用更大的 head_dim 来支持更大上下文。
```

## 六、旋转改变角度但不改变长度——为什么重要

### 6.1 长度代表什么

```
Q 向量的长度 = "我有多想找某类东西"（语义强度）

Q_猫 = [0.45, 0.32]   长度 ≈ 0.55  ← "猫"很想去找动词
Q_书 = [0.12, 0.08]   长度 ≈ 0.14  ← "书"没有特别想找的东西
```

**长度 = token 在自己角色上的置信度/强度。** 不同 token 的语义强度各不相同。

### 6.2 如果旋转改变长度会怎样

```
位置 p=0：Q_猫 长度 0.55
位置 p=100：Q_猫 长度 0.82 ← 变了！

同一个"猫"，在不同位置 Q 的长度不同
Q·K 结果变大不是因为"位置 100 的猫更想找动词"
而是旋转刚好让向量变长了
→ 模型分不清：这是位置造成的？还是语义造成的？❌
```

### 6.3 旋转不改变长度的好处

| 场景 | 不改变长度的作用 |
|:----|:----------------|
| **训练** | 语义强度（\|Q\|·\|K\|）和位置差（cos(差×freq)）**正交解耦**，各学各的 |
| **梯度** | 旋转是正交变换，梯度经过后长度不变，**更新量一致** |
| **多头** | 各头间长度比例**与位置无关**，分工不受干扰 |
| **推理** | 同个词出现在不同位置，**找东西的强度一样** |

**训练时的点积拆解：**

```
Q_rot · K_rot = |Q|·|K| × cos(旋转后夹角)
                ↑           ↑
             语义强度     位置差（纯靠 cos）

如果旋转改变长度：
  点积值 = (变了长度的|Q|)×(变了长度的|K|)×cos(夹角)
  → 分数高是因为"向量变长了"还是"真的想找"？分不清 ❌

如果旋转不改变长度：
  点积值 = 原来的|Q|×|K| × cos(差×freq)
          ↑纯语义强度↑   ↑纯位置差↑
  → 两者正交（互不干扰）→ 模型各学各的 ✅
```

**反向传播时的梯度稳定性：**

```
旋转矩阵是正交矩阵（cos/sin 构成的 2×2 矩阵），
对向量做正交变换 = 改变方向但不改变长度。

梯度经过正交变换后长度也不变：
  位置 0 的梯度大小 = 位置 100 的梯度大小 ✅
  → 同一个词在不同位置 receives 一致的 W_Q 更新
  → 训练稳定
```

### 6.4 本质原因

**cos 和 sin 构成的旋转矩阵是数学上的正交矩阵。** 性质：

```
旋转矩阵 R = [cosθ  -sinθ]
            [sinθ   cosθ]

|R|² = cos²θ + sin²θ = 1

所以旋转不改变向量长度，这是数学保证的，不是碰巧。
```

## 七、关键理解总结

| 问题 | 答案 |
|:----|:-----|
| 位置 p 是啥？ | token 在 seq 中的 index（0, 1, 2, ...） |
| Q 和 K 都旋转？ | 都转，角度一样。V 不转 |
| 不同头共用旋转？ | 共用频率表，但 W_Q/W_K 不同 |
| 旋转何时做？ | 点积之前，先旋转好 Q 和 K |
| KV Cache 存旋转后的 K？ | 是的 |
| cos 会重复吗？ | 单一 k 会，48 合一不重复 |
| 窗口扩展的本质？ | 调慢低频对的速度，让新位置角度落回已见范围 |
| 扩窗口有损失吗？ | 高频不动保近处精度，低频调慢远处略有模糊，总体可接受 |
