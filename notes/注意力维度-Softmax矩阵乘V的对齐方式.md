# 📐 注意力维度：Softmax(QK^T) @ V 的对齐方式

> Q: QK 算出来的「向量」怎么和 V 计算的？维度不一样？
> A: QK 算出来的不是「向量」，是「矩阵」——维度恰好对齐 V 的行数

---

## 核心一句话

**Q·K^T 的结果是 [seq_len × seq_len] 的注意力分数矩阵，不是向量。**
**这个矩阵的「列数」= token 个数，恰好等于 V 的「行数」，所以 @ V 维度对得上。**

---

## Step by Step 数字推演

句子"我 爱 你"（3个词），每个词 4 维（简化版）

### Step 1: Q、K、V 投影——维度不变

```
输入 x = [3 × 4]  (3个词, 每词4维)

Q = x × W_Q  →  [3 × 4]
K = x × W_K  →  [3 × 4]
V = x × W_V  →  [3 × 4]
```

Q、K、V 各自独立投影，但输入相同，所以维度一样。

### Step 2: Q @ K^T——结果从 [3×4] 变成 [3×3]

```
Q_我 = [0.48, 0.42, 0.32, 0.51]       K_我 = [0.55, 0.38, 0.29, 0.46]
Q_爱 = [0.27, 0.41, 0.29, 0.33]       K_爱 = [0.41, 0.32, 0.28, 0.43]
Q_你 = [0.35, 0.44, 0.39, 0.38]       K_你 = [0.42, 0.40, 0.33, 0.41]

Q(K^T) — K 转置后竖着排：
            K_我^T  K_爱^T  K_你^T
           ┌────────────────────────┐
Q_我 ─────│ 0.752  0.640  0.685  │  ← "我"去看每个人
Q_爱 ─────│ 0.582  0.541  0.601  │  ← "爱"去看每个人
Q_你 ─────│ 0.663  0.598  0.617  │  ← "你"去看每个人
           └────────────────────────┘
           [3×3] ✅ — 不是向量，是矩阵！
```

### Step 3: Softmax 按行做——还是 [3×3]

```
"我"行：softmax([0.752, 0.640, 0.685]) → [0.343, 0.325, 0.332]
"爱"行：softmax([0.582, 0.541, 0.601]) → [0.335, 0.324, 0.341]
"你"行：softmax([0.663, 0.598, 0.617]) → [0.334, 0.330, 0.336]

→ 每行加起来=1，还是 [3×3]
```

### Step 4: @ V——[3×3] @ [3×4] = [3×4]

```
softmax 矩阵：                V 矩阵：
[0.343, 0.325, 0.332]    @   V_我 = [0.33, 0.52, 0.41, 0.37]
[0.335, 0.324, 0.341]        V_爱 = [0.45, 0.28, 0.51, 0.39]
[0.334, 0.330, 0.336]        V_你 = [0.36, 0.43, 0.38, 0.42]

对齐的是高亮的部分 👆 — 两个 "3" 都是 token 个数

"我"的新向量 = 0.343×V_我 + 0.325×V_爱 + 0.332×V_你
            = [0.379, 0.412, 0.433, 0.393]      ← 4个数

"爱"的新向量 = 0.335×V_我 + 0.324×V_爱 + 0.341×V_你
            = [0.381, 0.408, 0.435, 0.394]      ← 4个数

"你"的新向量 = 0.334×V_我 + 0.330×V_爱 + 0.336×V_你
            = [0.374, 0.416, 0.428, 0.389]      ← 4个数

最终 = [3×4] ✅ 和输入一样！
```

## 维度对照总表

| 步骤 | 形状 | 说明 |
|:----|:----|:----|
| 输入 x | [seq, d] | seq=token数, d=向量维度 |
| Q = x·W_Q | [seq, d] | 每个词的 Query |
| K = x·W_K | [seq, d] | 每个词的 Key |
| V = x·W_V | [seq, d] | 每个词的 Value |
| Q·K^T | **[seq, seq]** | 注意力分数矩阵——所有配对的点积 |
| softmax(Q·K^T) | [seq, seq] | 每行归一化到 0~1 |
| **softmax(Q·K^T) @ V** | **[seq, d]** | 加权平均——对齐的是 seq（位置个数） |

## minimind 实际代码对照

```python
scores = (xq @ xk.transpose(-2, -1)) / math.sqrt(self.head_dim)
# xq: [bsz, n_heads, seq, head_dim]
# xk.transpose(-2,-1): [bsz, n_heads, head_dim, seq]
# scores: [bsz, n_heads, seq, seq]

output = F.softmax(scores, dim=-1) @ xv
# softmax(scores): [bsz, n_heads, seq, seq]
# xv: [bsz, n_heads, seq, head_dim]
# output: [bsz, n_heads, seq, head_dim]
```

注意 flip 了 permute 的顺序。所以符号和笔记里的 seq/d 对应关系是：
- 代码里的 `seq` = 这里的 token 数（我、爱、你）
- 代码里的 `head_dim` = 这里的向量维度（4）

---

## 关键理解

**Attention 的计算过程本质上是两次矩阵乘法：**

```
第一次：Q @ K^T   →  每个词和所有词算相似度 →  [seq × seq]
第二次：权重 @ V   →  每个词用权重混合所有 V →  [seq × d]
```

中间不需要「维度匹配」的担忧——第一次 Q 和 K 的 `d` 匹配（因为投影后的 Q 和 K 维度相同），第二次 `seq` 匹配（因为是按位置对齐）。中间的 [seq × seq] 矩阵就是「注意力桥」。
