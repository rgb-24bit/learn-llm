## 第一部分：前置数学知识

### 1.1 向量是什么？

**向量就是一串排好的数字**。比如：
```
q = [3, 1, 4]     ← 一个 3 维向量
k = [2, 7, 1]     ← 另一个 3 维向量
```

可以想象成**三维空间中的一个箭头**：
```
   z
   ↑
   │   · (3, 1, 4)
   │  /
   │ /
   └──────→ y
  /
 /
x
```

### 1.2 点积（Dot Product）——最基本操作

两个向量的点积 = **对应位置相乘，再全部加起来**：

```
q = [3, 1, 4]
k = [2, 7, 1]

q·k = 3×2 + 1×7 + 4×1
    = 6 + 7 + 4
    = 17
```

**点积越大，两个向量越「相似」**（方向越接近）：
```
q = [1, 0, 0]   ← 指向 x 轴正方向
k = [1, 0, 0]   ← 也是 x 轴正方向
q·k = 1 → 方向相同，点积最大

q = [1, 0, 0]
k = [-1, 0, 0]  ← 反方向
q·k = -1 → 方向相反，点积最小

q = [1, 0, 0]
k = [0, 1, 0]   ← 垂直方向
q·k = 0 → 不相关
```

点积的取值范围：从 -1 到 1（当向量长度为 1 时，也就是"单位向量"）

### 1.3 矩阵乘向量

**矩阵 = 一张数字表格**。矩阵 × 向量 = 把矩阵的每一行和向量做点积：

```
W = | 0.5  -0.2  0.1 |    ← 3 行 3 列的矩阵
    | 0.3   0.4  0.0 |
    | 0.1   0.2  0.6 |

x = [1, 2, 3]               ← 3 维向量

W × x = 第一行·x , 第二行·x , 第三行·x

第一行·x = 0.5×1 + (-0.2)×2 + 0.1×3 = 0.5 - 0.4 + 0.3 = 0.4
第二行·x = 0.3×1 + 0.4×2 + 0.0×3 = 0.3 + 0.8 + 0 = 1.1
第三行·x = 0.1×1 + 0.2×2 + 0.6×3 = 0.1 + 0.4 + 1.8 = 2.3

结果 = [0.4, 1.1, 2.3]
```

**矩阵 × 向量 = 向量的重新组合**。W 的每一行决定"从 x 里取多少"。

### 1.4 Softmax——把一堆数字变成"概率"

```
输入：  [2.0, 1.0, 0.1]    ← 任意数字，可正可负
目标：  [p₁, p₂, p₃]       ← 都大于 0 且加起来 = 1

步骤 1：对每个数取 e^x（e≈2.718）：
  e^2.0 = 7.39
  e^1.0 = 2.72
  e^0.1 = 1.11

步骤 2：求和 = 7.39 + 2.72 + 1.11 = 11.22

步骤 3：每个除以总和：
  p₁ = 7.39/11.22 = 0.66
  p₂ = 2.72/11.22 = 0.24
  p₃ = 1.11/11.22 = 0.10

结果： [0.66, 0.24, 0.10]    ← 加起来 = 1 ✅
```

效果：**大数变得更大，小数变得更小**，但总体还是"概率"。

## 第二部分：QKV 的数学原理

### 2.1 三句话总结 QKV

```
每个位置 x_i：
  乘 W_Q → Q_i = "我在找什么？"（查询）
  乘 W_K → K_i = "我有什么？"（标签）
  乘 W_V → V_i = "我能提供什么？"（内容）
```

### 2.2 投影矩阵 W_Q, W_K, W_V

假设每个位置的向量是 **768 维**，我们要投影到的空间也是 **768 维**：

```
W_Q = 一个 768 行 × 768 列的矩阵
      （约 590,000 个可学习参数）

x = [0.2, -0.5, 0.8, ..., 0.1]   ← 768 个数

Q = W_Q × x

Q = [0.4, -0.1, ..., 0.7]        ← 也是 768 个数
```

**W_Q 的每一行都是一个"过滤器"**：
```
行 0：检查 x 里是否包含"我在做查询"的信号
行 1：检查 x 里是否包含"我要找名词"的信号
行 2：检查 x 里是否包含"我要找动词"的信号
...
W_K 的每一行也是类似的过滤器，但关注的是"展示自己有什么"
W_V 的每一行关注的是"提供什么有价值的信息"
```

W_Q, W_K, W_V 在学习过程中各自学会了不同角色。**训练开始时是随机的，训练结束时它们学会了分工。**

### 2.3 注意力分数计算

```
序列有 4 个位置： ["我", "爱", "你", "。"]

每个位置有自己的 Q、K、V：

        Q              K              V
我  →  [0.2, ...]    [0.5, ...]    [0.1, ...]
爱  →  [0.8, ...]    [0.3, ...]    [0.9, ...]
你  →  [0.1, ...]    [0.6, ...]    [0.4, ...]
。  →  [0.0, ...]    [0.2, ...]    [0.0, ...]

注意力分数矩阵 = Q × K^T（所有 Q 和所有 K 做点积）

           Q(我)·K(我)   Q(我)·K(爱)   Q(我)·K(你)   Q(我)·K(。)
           Q(爱)·K(我)   Q(爱)·K(爱)   Q(爱)·K(你)   Q(爱)·K(。)
scores =   Q(你)·K(我)   Q(你)·K(爱)   Q(你)·K(你)   Q(你)·K(。)
           Q(。)·K(我)   Q(。)·K(爱)   Q(。)·K(你)   Q(。)·K(。)

比如 Q(爱)·K(你) 很高 → "爱"在查"对象"，"你"正好有"我是对象"的标签
```

### 2.4 缩放（除以 √d）

```python
scores = Q @ K^T / √dim
```

**为什么除以 √dim？** 因为当 dim 很大时（768），点积的数值会变得很大（因为把 768 个乘积加在一起）。大数值会导致 softmax 后"赢家通吃"（一个接近 1，其他接近 0），模型无法学到精细的注意力分配。

除以 √768 ≈ 27.7，把数值拉回到合理范围。

### 2.5 加权求和——最终输出

```
attention_weights = softmax(scores)  
# 每行加起来 = 1，表示一个位置对其他位置的关注度

output_我 = attention_weights[我,我]×V_我 
          + attention_weights[我,爱]×V_爱 
          + attention_weights[我,你]×V_你 
          + attention_weights[我,。]×V_。

# "我"的输出 = "我"自己的信息 × 关注自己程度
#             + "爱"的信息 × 关注"爱"的程度
#             + ...
```

**每层的输出长度和输入一样**（768 维），但每个位置的向量已经融合了其他位置的信息。

### 2.6 多头注意力

8 个头的意思是：**8 组独立的 W_Q, W_K, W_V**，分别关注不同类型的关系：

```
头 1 的 W_Q: 学会检查"我在找语法主语"
头 1 的 W_K: 学会展示"我是名词"
→ 头 1 学会主谓关系

头 2 的 W_Q: 学会检查"这个代词指代什么"
头 2 的 W_K: 学会展示"我是指代对象"
→ 头 2 学会计代关系

...

每个头的输出是 96 维（768/8=96），拼回 768 维
```

## 第三部分：RoPE 的数学原理

### 3.1 前置知识：三角函数

```
在直角三角形中：
     ↑
     │
   h │   ↗ θ     ← h 是斜边
     │
     └──────→
        a

sin θ = 对边/h
cos θ = 邻边/a
```

**在单位圆上**（半径=1）：
```
      ↑ y
      │
   ───┤───→ x
      │
 
角度 θ 在圆上对应的点 = (cos θ, sin θ)

θ=0° → (1, 0)    ← 指向右
θ=90° → (0, 1)    ← 指向上
θ=180° → (-1, 0)  ← 指向左
θ=360° → (1, 0)    ← 回到起点
```

**关键性质**：
- cos(θ) 和 sin(θ) 的取值范围都是 [-1, 1]
- cos²θ + sin²θ = 1（勾股定理）
- cos 和 sin 是周期函数：cos(θ+2π)=cos(θ)（每 360° 重复一次）

### 3.2 二维旋转

二维向量 (x, y) 旋转角度 θ：

```
旋转前的点： (x, y)           ← 比如 (1, 0)，指向右
旋转 θ 度后： (x', y')        ← 比如转 90° → (0, 1)

公式：
x' = x·cos θ - y·sin θ
y' = x·sin θ + y·cos θ
```

**具体算：**

```
旋转 (1, 0) 角度 θ=90°：
cos 90° = 0, sin 90° = 1

x' = 1×0 - 0×1 = 0
y' = 1×1 + 0×0 = 1

结果 = (0, 1) ✅ 指向上方

旋转 (1, 0) 角度 θ=45°：
cos 45° ≈ 0.707, sin 45° ≈ 0.707

x' = 1×0.707 - 0×0.707 = 0.707
y' = 1×0.707 + 0×0.707 = 0.707

结果 ≈ (0.707, 0.707) ✅ 指向右上角
```

**性质：旋转不改变向量的长度**（模长不变）：
```
旋转前：1²+0²=1 → 长度=1
旋转后：0.707²+0.707²=0.5+0.5=1 → 长度=1 ✅
```

### 3.3 RoPE 的核心：用旋转来编码位置

**基本思路**：每个位置 p 把 Q 和 K 旋转 p×ω 度（ω 是频率）：
```
位置 0 的 Q：旋转 0°    ← 不转
位置 1 的 Q：旋转 ω°    ← 转一点
位置 2 的 Q：旋转 2ω°   ← 转更多
```

**问题**：Q 和 K 都是 768 维的，不能整个向量一起旋转（那会搞乱内部结构）。

**解决方案**：每两个维度绑在一起旋转，不同对用不同的旋转速度。

### 3.4 维度对：为什么是两两一组？

二维旋转需要两个数。看公式：
```
x' = x·cosθ - y·sinθ    ← 新 x 需要原来的 x 和 y
y' = x·sinθ + y·cosθ    ← 新 y 也需要原来的 x 和 y
```

所以要旋转二维空间，需要一对维度 (d_i, d_j)。一个维度自己没法转。

**RoPE 的配对方式**：前半和后半配对

```
dim=768 的向量，每头 96 维：

头 1 的 96 维向量：
[ d₀, d₁, d₂, ..., d₄₇ | d₄₈, d₄₉, ..., d₉₅ ]
  ↑                      ↑
  └── 一对 (共用 ω₀) ──┘
       ↑              ↑
       └── 一对 ─────┘ (共用 ω₁)

对 0：(d₀, d₄₈) 共用频率 ω₀
对 1：(d₁, d₄₉) 共用频率 ω₁
对 2：(d₂, d₅₀) 共用频率 ω₂
...
对 47：(d₄₇, d₉₅) 共用频率 ω₄₇
```

**为什么这样配对？** 因为 rotate_half 函数是切成前后两半再交叉的。这种配对方式让计算特别高效——一次 torch.cat 就完成了所有对的 90° 旋转准备。

### 3.5 频率怎么算？

```python
ω_i = 1 / 10000^(2i/dim)
```

dim=96，所以 i=0,2,4,...,94（步长 2，共 48 个频率）：

```
对 0  (2i=0):   ω₀ = 1/10000^0        = 1.0000   ← 最快
对 1  (2i=2):   ω₁ = 1/10000^(2/96)   ≈ 0.9306
对 2  (2i=4):   ω₂ = 1/10000^(4/96)   ≈ 0.8659
...
对 24 (2i=48):  ω₂₄= 1/10000^(48/96)  = 0.3162   ← 中速
...
对 47 (2i=94):  ω₄₇= 1/10000^(94/96)  ≈ 0.1069   ← 最慢
```

**频率从 1.0 到 0.1，覆盖了一个范围。这就是之前 NTK 里说的频率分布！**

### 3.6 每个位置 p 的旋转角度

```
对 0（最快）：位置 p 的旋转角度 = p × 1.0
对 24（中速）：位置 p 的旋转角度 = p × 0.316
对 47（最慢）：位置 p 的旋转角度 = p × 0.107

位置 5 时：
对 0 转了 5×1=5 rad≈286°
对 24 转了 5×0.316=1.58 rad≈91°
对 47 转了 5×0.107=0.54 rad≈31°
```

### 3.7 每对维度的旋转（具体算一例）

对一对 (d, d+48)，旋转角度 φ = p × ω_d：

```
旋转前：q_d = 1.0, q_{d+48} = 0.5
频率 ω_d = 0.316（对 24）
位置 p = 5

φ = 5 × 0.316 = 1.58 rad

cos(1.58) ≈ -0.002
sin(1.58) ≈ 1.000

旋转后：
q'_d      = 1.0 × (-0.002) + (-0.5) × 1.000 = -0.002 - 0.5 = -0.502
q'_{d+48} = 0.5 × (-0.002) + 1.0 × 1.000   = -0.001 + 1.0 = 0.999

点 (1.0, 0.5) → 旋转 91° → (-0.502, 0.999)

验证长度不变：√(1²+0.5²)=√1.25=1.118
            √((-0.502)²+0.999²)=√(0.252+0.998)=√1.25=1.118 ✅
```

### 3.8 为什么这样做点积就能知道位置差？

这是 RoPE 最精妙的部分，我们一步步推导。

先看**只有一对维度**的简化情况（2 维）：

```
Q 在位置 m，旋转角度 mω
K 在位置 n，旋转角度 nω

Q_rot = (q₀cos(mω) - q₁sin(mω), q₁cos(mω) + q₀sin(mω))
K_rot = (k₀cos(nω) - k₁sin(nω), k₁cos(nω) + k₀sin(nω))
```

现在做点积 Q_rot·K_rot：

**步骤 1：第一项 × 第一项**
```
q₀cos(mω) - q₁sin(mω)   乘上   k₀cos(nω) - k₁sin(nω)
= q₀k₀·cos(mω)cos(nω)        ← q₀k₀ 乘两个 cos 相乘
- q₀k₁·cos(mω)sin(nω)        ← q₀k₁ 乘 cos×sin
- q₁k₀·sin(mω)cos(nω)        ← q₁k₀ 乘 sin×cos
+ q₁k₁·sin(mω)sin(nω)        ← q₁k₁ 乘两个 sin 相乘
```

**步骤 2：第二项 × 第二项**
```
q₁cos(mω) + q₀sin(mω)   乘上   k₁cos(nω) + k₀sin(nω)
= q₁k₁·cos(mω)cos(nω)
+ q₁k₀·cos(mω)sin(nω)
+ q₀k₁·sin(mω)cos(nω)
+ q₀k₀·sin(mω)sin(nω)
```

**步骤 3：把 8 项合在一起**

先找 q₀k₀ 的项（来自第一项×第一项 和 第二项×第二项）：
```
q₀k₀·cos(mω)cos(nω) + q₀k₀·sin(mω)sin(nω)
= q₀k₀·[cos(mω)cos(nω) + sin(mω)sin(nω)]
```

用三角恒等式 **cos(A-B) = cosA·cosB + sinA·sinB**：
```
= q₀k₀·cos(mω - nω)
= q₀k₀·cos((m-n)ω)
```

再找 q₁k₁ 的项：
```
q₁k₁·sin(mω)sin(nω) + q₁k₁·cos(mω)cos(nω)
= q₁k₁·[sin(mω)sin(nω) + cos(mω)cos(nω)]
= q₁k₁·cos((m-n)ω)
```

**说明**：第 2 项和第 3 项的等式用了 **cos(A-B) = cosA·cosB + sinA·sinB**，这是三角恒等式，只要把 A=mω, B=nω 代入，就能得到 cos(mω-nω)=cos((m-n)ω)。

再找混合项 q₀k₁ 和 q₁k₀：
```
来自第一项：-q₀k₁·cos(mω)sin(nω) - q₁k₀·sin(mω)cos(nω)
来自第二项：+q₀k₁·sin(mω)cos(nω) + q₁k₀·cos(mω)sin(nω)
```

**注意 -q₀k₁·cos(mω)sin(nω) 和 +q₀k₁·sin(mω)cos(nω)** 不能抵消。用三角恒等式 **sin(A-B) = sinA·cosB - cosA·sinB**：

```
q₀k₁·[sin(mω)cos(nω) - cos(mω)sin(nω)] = q₀k₁·sin((m-n)ω)
q₁k₀·[cos(mω)sin(nω) - sin(mω)cos(nω)] = -q₁k₀·sin((m-n)ω)
```

**步骤 4：最终结果**

```
Q_rot·K_rot = (q₀k₀+q₁k₁)×cos((m-n)ω) + (q₀k₁-q₁k₀)×sin((m-n)ω)

第一部分：(Q·K) × cos(Δpos·ω)  
  → 语义相似度 × 位置差的余弦

第二部分：(Q×K) × sin(Δpos·ω)
  → 语义垂直度 × 位置差的正弦
```

### 3.9 几何意义

第一部分好理解：**位置差为 0 时 cos=1，原始点积不变；位置差越大 cos 越小，点积衰减。**

那第二部分呢？(q₀k₁-q₁k₀) 在几何上等于**两个向量构成的平行四边形的有向面积**（2D 叉积的大小）：

```
如果 q 和 k 方向相同：q₀k₁-q₁k₀=0，第二项为零
如果 q 和 k 垂直：q₀k₁-q₁k₀ 最大，第二项贡献最大

sin((m-n)ω) 在 Δpos=0 时=0，Δpos 增大时先增后减
```

**合起来的效果**：位置差不仅通过 cos 衰减语义相似度，还通过 sin 引入了"垂直度×位置"的附加信息。这增加了位置编码的唯一性——即使两个位置的 cos 值相同，sin 值可能不同。

### 3.10 扩展到 48 对维度

每个维度对有自己的频率 ω_i，对所有对求和：

```
Q_rot·K_rot = Σ_{i=0}^{47} (Q_i·K_i)×cos(Δpos·ω_i) 
             + (Q_i×K_i)×sin(Δpos·ω_i)
```

不同的 ω_i 导致不同的衰减速度：
```
ω₀=1.0:   位置差 1 时 cos(1)=0.54, sin(1)=0.84  → 变化剧烈
ω₂₄=0.316:位置差 1 时 cos(0.316)=0.95, sin(0.316)=0.31 → 平缓变化
ω₄₇=0.107:位置差 1 时 cos(0.107)=0.99, sin(0.107)=0.11 → 几乎不变
```

**整体效果**：
- **局部位置**：高频维度（ω≈1）的 cos 快速下降，sin 快速变化 → 模型能精细区分邻近位置
- **远距离**：低频维度（ω≈0.1）的 cos 缓慢下降 → 模型仍能感知大致距离
- **所有频率叠加** → 每个位置差 Δpos 对应一个**唯一的 96 维 cos+sin 模式** → 模型通过识别这个模式来知道距离

### 3.11 RoPE 的完整计算流程

```python
# 1. 预计算（做一次，推理时直接用）
precompute_freqs_cis(dim=96, end=32768):
  生成 48 个频率 ω₀, ω₁, ..., ω₄₇
  对所有位置 p=0..32767:
    cos[p] = [cos(pω₀), cos(pω₁), ..., cos(pω₄₇)] × 2
    sin[p] = [sin(pω₀), sin(pω₁), ..., sin(pω₄₇)] × 2

# 2. 在 Attention 中应用
def attention(x, cos, sin):
    Q = x @ W_Q    # [batch, seq, 96]
    K = x @ W_K    # [batch, seq, 96]
    V = x @ W_V    # [batch, seq, 96]
    
    # 取当前位置对应的 cos/sin
    cos_p = cos[position_ids]   # [batch, seq, 96]
    sin_p = sin[position_ids]
    
    # 应用旋转到 Q 和 K
    Q_rot = Q × cos_p + rotate_half(Q) × sin_p
    K_rot = K × cos_p + rotate_half(K) × sin_p
    
    # 带位置的注意力计算
    scores = Q_rot @ K_rot^T / √96
    # scores[i][j] = (Q_i·K_j)的旋转版本
    #              = Σ(Q_i·K_j的每对)×cos((i-j)·ω) + ...
    # 分数被位置差 (i-j) 调制
    
    return softmax(scores) @ V
```
