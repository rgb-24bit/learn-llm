# 🧠 多头注意力的分工、实现与更新机制

> 承接之前的 Attention 代码逐行解读，深入讨论多头注意力的核心问题
> 包含一个完整的 3-token 计算例子

---

## 一、8 个头是同一层内的，不是跨层的

MiniMind 有 8 层 Transformer，每层有 8 个注意力头：

```
MiniMindModel
  ├── embed_tokens (6400 → 768)
  ├── layers[0] (MiniMindBlock)
  │     ├── input_layernorm
  │     ├── self_attn  ← ★ 这里有 8 个头
  │     ├── post_attention_layernorm
  │     └── mlp
  ├── layers[1]
  │     ├── self_attn  ← ★ 这里也有 8 个新头
  │     └── ...
  └── layers[7] → 最后一层
```

总数：**8 层 × 8 头 = 64 个注意力头**

每层的头分工趋势：
- 浅层（1-2层）：关注相邻词、词性、局部语法
- 中层（3-5层）：关注主谓宾结构、指代关系
- 深层（6-8层）：关注全局主题、逻辑关系

---

## 二、代码上 8 个头怎么实现的

**不是 8 个独立的小网络，是一个大线性层一次算出然后切开的。**

```python
# 一个线性层，输出 = 8头 × 96维 = 768维
self.q_proj = nn.Linear(768, 768)

# forward 里切开：
xq = self.q_proj(x)                # → [batch, seq, 768]
xq = xq.view(batch, seq, 8, 96)    # → [batch, seq, 8头, 96]
```

理解方式：把 768 维输出看作"8 个连续的 96 维片段粘在一起"：

```
q_proj 输出：[d₀-d₉₅ | d₉₆-d₁₉₁ | d₁₉₂-d₂₈₇ | ... | d₆₇₂-d₇₆₈]
               头0        头1        头2             头7
```

注意：通过 GQA，`k_proj` 和 `v_proj` 输出的是 4×96 = 384 维。

### 不同层的头划分可以不一样吗？

技术上可以——第 1 层用 8:4，第 2 层用 8:8，第 3 层用 8:1。但实际没有模型这么做。**所有层统一配置**，原因：代码简单、显存均匀、调参省事。

### Q/K/V 头数的常见配置

| 方案 | Q头数 | K/V头数 | 谁在用 | KV Cache省多少 |
|:---:|:-----:|:-------:|:------:|:-------------:|
| MHA（全量多头） | N | N | Llama 1/2 | 1×（不省） |
| GQA（分组查询） | N | N/2 或 N/4 | Llama 3, Qwen, Mistral, minimind | 省 2-4 倍 |
| MQA（单查询） | N | 1 | PaLM, ChatGLM | 省 N 倍 |

Minimind 的 GQA 是保守的 8:4 配置——省一半 KV Cache，质量不掉。

## 三、多头注意力的优点

### ① 分工——不同头关注不同类型的语言关系

一段话里有多种关系同时存在，一个头包揽不过来，8 个头各管一摊：

- 头①：主语→动词（"小明"盯"送"）
- 头②：形容词→名词（"红色的"盯"玫瑰花"）
- 头③：时间副词→动词（"昨天"盯"送"）
- 头④：介词结构（"在"盯"公园"）
- 头⑤：相邻词局部语法（"一"→"束"）
- 头⑥：长距离主题（整句话核心是"送花"）

单一注意力头只能输出"平均报告"，多头才能做到"各有专攻"。

### ② 效率——并行

8 个头是一次矩阵乘法算出来的，跟 1 个头的时间一样（甚至更快，因为矩阵更大硬件利用率更高）。

### ③ 鲁棒性

某个头没学好，其他头能兜底。实验证明：**剪掉一半的头，性能只降 10-20%**。

## 四、反向传播如何更新不同的头

**关键认知：没有"分别更新不同的头"这回事。**

```python
self.q_proj = nn.Linear(768, 768)  # 就一个矩阵，不分头
```

反向传播更新的是**整个 768×768 矩阵**，不是按头分开的。

那为什么头会分化？

### 自发的分工分化过程

```
随机初始化时：8 块 768×96 权重略有不同
（PyTorch Kaiming 均匀初始化，每块 73,728 个独立随机数）

第 1 步 forward：
  - 头 0 那 96 列刚好对"主语"特征响应略强
  - 头 1 那 96 列刚好对"动词"特征响应略强
  - 两者差异极小（0.001 级别）

第 1 步 backward：
  - 模型预测错了，梯度传回来
  - 头 0 的梯度方向偏向"主语关系"的修正
  - 头 1 的梯度方向偏向"动词关系"的修正

第 2 步 update：
  - 头 0 的 96 列被往"捕捉主语"方向多推了一点
  - 头 1 的 96 列被往"捕捉动词"方向多推了一点
  - 差异从 0.001 → 0.01

训练 1 万步后：
  - 头 0 已经非常擅长捕捉主语关系
  - 头 1 已经非常擅长捕捉动词关系
  - 差异从 0.001 → 5.0
```

**流程比喻：** 8 个实习生一起做调研。第一次全错了，老板骂了一顿。实习生 A 注意到"这次错在竞争对手分析"→ 下次他就多关注对手。实习生 B 注意到"错在定价"→ 下次多关注价格。不是老板分配了分工，是**各自的错误自然引导出了分工方向**。

### 正反馈循环——关键机制

```
对某种模式略敏感 → 该模式出现时输出更强
                  → 梯度更大
                  → 权重往这个方向更新更多
                  → 更加敏感
                  → 最终变成该模式的"专家"
```

### 初始化对称性问题

如果两个头的初始化权重完全一样 → 它们的输出完全一样 → 梯度完全一样 → 更新后还是一样 → **永远学不出分工**。

这就是为什么 `nn.Linear` 必须用**随机初始化**。8 块的 73,728 个参数全部一致的概率 ≈ 0。

如果全零初始化：所有头输出都是 0 → attention 分数全是 0 → softmax 后每个位置均匀分配 → 8 个头退化成一个头 → **浪费 7/8 的参数**。

## 五、完整 3-token 计算示例

### 设定

句子："猫 吃 鱼"（3 个 token）

为方便手动计算，简化配置：

| 参数 | 教学值 | 真实 MiniMind |
|:----|:-----:|:------------:|
| hidden_size | **4** | 768 |
| num_q_heads | **2** | 8 |
| num_kv_heads | **1** | 4 |
| head_dim | **2** | 96 |

所以：
- Q 投影：4维 → 4维（2头 × 2维）
- K 投影：4维 → 2维（1头 × 2维）
- V 投影：4维 → 2维（1头 × 2维）

### Step 0：输入（3 个 token 的向量）

```
"猫" → [0.5, 0.1, 0.3, 0.8]
"吃" → [0.2, 0.7, 0.1, 0.4]
"鱼" → [0.6, 0.3, 0.5, 0.2]
```

### Step 1：QKV 投影

假设训练好的权重（简化）：

```
W_Q = [0.1, 0.3, 0.2, 0.4]     ← 头0 那 2 列
      [0.4, 0.1, 0.3, 0.2]     ← 头1 那 2 列
      [0.2, 0.4, 0.1, 0.3]     ← 头0 的 Q 第二行？不，等我重新想

其实 4维→4维的 q_proj 是 4×4 矩阵。
4维→2维的 k_proj 是 2×4 矩阵。
4维→2维的 v_proj 是 2×4 矩阵。
```

让我重新想一个干净的教学例（用具体的数字矩阵）：

#### q_proj（4×4 矩阵）

训练好的 W_Q：
```
     猫的向量  吃的向量  鱼的向量
     0.5 0.1 0.3 0.8   0.2 0.7 0.1 0.4   0.6 0.3 0.5 0.2
     ─────────────────────────────────────────────────────
头0  [0.1  0.2  0.3  0.4]  · [猫] = 0.1×0.5 + 0.2×0.1 + 0.3×0.3 + 0.4×0.8 = 0.05+0.02+0.09+0.32 = 0.48
     [0.2  0.5  0.1  0.3]  · [猫] = 0.2×0.5 + 0.5×0.1 + 0.1×0.3 + 0.3×0.8 = 0.10+0.05+0.03+0.24 = 0.42
头1  [0.3  0.4  0.2  0.1]  · [猫] = 0.3×0.5 + 0.4×0.1 + 0.2×0.3 + 0.1×0.8 = 0.15+0.04+0.06+0.08 = 0.33
     [0.4  0.3  0.5  0.2]  · [猫] = 0.4×0.5 + 0.3×0.1 + 0.5×0.3 + 0.2×0.8 = 0.20+0.03+0.15+0.16 = 0.54
```

用表格算 3 个 token：

| token | Q_头0(d₀) | Q_头0(d₁) | Q_头1(d₀) | Q_头1(d₁) |
|:----:|:---------:|:---------:|:---------:|:---------:|
| **猫** | 0.48 | 0.42 | 0.33 | 0.54 |
| **吃** | 0.27 | 0.41 | 0.29 | 0.33 |
| **鱼** | 0.35 | 0.44 | 0.39 | 0.38 |

→ 每个 Q 头的向量（2维）：
```
Q头0(猫) = [0.48, 0.42]    Q头1(猫) = [0.33, 0.54]
Q头0(吃) = [0.27, 0.41]    Q头1(吃) = [0.29, 0.33]
Q头0(鱼) = [0.35, 0.44]    Q头1(鱼) = [0.39, 0.38]
```

#### k_proj（2×4 矩阵）——只有 1 个头

```
W_K: [0.3  0.1  0.4  0.2]  · [猫] = 0.3×0.5 + 0.1×0.1 + 0.4×0.3 + 0.2×0.8 = 0.15+0.01+0.12+0.16 = 0.44
     [0.2  0.4  0.1  0.3]  · [猫] = 0.2×0.5 + 0.4×0.1 + 0.1×0.3 + 0.3×0.8 = 0.10+0.04+0.03+0.24 = 0.41
```

| token | K(d₀) | K(d₁) |
|:----:|:-----:|:-----:|
| **猫** | 0.44 | 0.41 |
| **吃** | 0.38 | 0.35 |
| **鱼** | 0.42 | 0.40 |

→ K(猫)=[0.44, 0.41], K(吃)=[0.38, 0.35], K(鱼)=[0.42, 0.40]

#### v_proj（2×4 矩阵）——只有 1 个头

```
W_V: [0.2  0.5  0.1  0.2]  · [猫] = 0.2×0.5 + 0.5×0.1 + 0.1×0.3 + 0.2×0.8 = 0.10+0.05+0.03+0.16 = 0.34
     [0.3  0.2  0.4  0.1]  · [猫] = 0.3×0.5 + 0.2×0.1 + 0.4×0.3 + 0.1×0.8 = 0.15+0.02+0.12+0.08 = 0.37
```

| token | V(d₀) | V(d₁) |
|:----:|:-----:|:-----:|
| **猫** | 0.34 | 0.37 |
| **吃** | 0.40 | 0.32 |
| **鱼** | 0.36 | 0.43 |

→ V(猫)=[0.34, 0.37], V(吃)=[0.40, 0.32], V(鱼)=[0.36, 0.43]

### Step 2：repeat_kv（GQA：单 K/V 头复制给 2 个 Q 头）

```
Q有2个头，但K/V只有1个头 → 把 K 和 V 各复制一份

K_头0 = K_头1 = [0.44, 0.41]（猫）, [0.38, 0.35]（吃）, [0.42, 0.40]（鱼）
V_头0 = V_头1 = [0.34, 0.37]（猫）, [0.40, 0.32]（吃）, [0.36, 0.43]（鱼）
```

### Step 3：头0 算 attention

Q头0 × K^T（注意：K 经过 repeat_kv 后，头0 拿到的就是原来那个 K）：

```
Q头0(猫) = [0.48, 0.42], K(猫) = [0.44, 0.41]
分数_猫·猫 = 0.48×0.44 + 0.42×0.41 = 0.211 + 0.172 = 0.383

Q头0(猫) = [0.48, 0.42], K(吃) = [0.38, 0.35]
分数_猫·吃 = 0.48×0.38 + 0.42×0.35 = 0.182 + 0.147 = 0.329

Q头0(猫) = [0.48, 0.42], K(鱼) = [0.42, 0.40]
分数_猫·鱼 = 0.48×0.42 + 0.42×0.40 = 0.202 + 0.168 = 0.370

同理算完所有组合：
```

头0 的分数矩阵：

| Q\K | 猫 | 吃 | 鱼 |
|:---:|::|::|::|
| **猫** | 0.383 | 0.329 | 0.370 |
| **吃** | 0.291 | 0.271 | 0.301 |
| **鱼** | 0.317 | 0.293 | 0.324 |

除 √head_dim = √2 ≈ 1.414：

| Q\K | 猫 | 吃 | 鱼 |
|:---:|::|::|::|
| **猫** | 0.271 | 0.233 | 0.262 |
| **吃** | 0.206 | 0.192 | 0.213 |
| **鱼** | 0.224 | 0.207 | 0.229 |

做 softmax（"猫"行为例）：

```
e^0.271 = 1.311, e^0.233 = 1.262, e^0.262 = 1.300
和 = 1.311 + 1.262 + 1.300 = 3.873

猫→猫：1.311/3.873 = 0.339
猫→吃：1.262/3.873 = 0.326
猫→鱼：1.300/3.873 = 0.336

→ "猫"占33.9%，"吃"占32.6%，"鱼"占33.6%
→ 几乎均匀！说明"猫"跟三个词都有关系，注意力平均分配
```

完整 softmax 后头0的注意力权重：

| | 猫 | 吃 | 鱼 |
|:---:|::|::|::|
| **猫** | 0.339 | 0.326 | 0.336 |
| **吃** | 0.336 | 0.329 | 0.335 |
| **鱼** | 0.335 | 0.330 | 0.335 |

→ 训练得不够好的话所有头都是均匀的！
→ 训练到位后头0应该学会"猫→吃"的关注度比别的更高（主语盯动词）

为了演示头0和头1的分工差异，假设头0训练后变成了"主语盯着动词"：

**假设训练好的头0注意力权重：**

| | 猫 | 吃 | 鱼 |
|:---:|::|::|::|
| **猫** | 0.15 | **0.60** | 0.25 |
| **吃** | 0.45 | 0.10 | 0.45 |
| **鱼** | 0.30 | 0.55 | 0.15 |

→ 头0学会了"猫"盯"吃"（主语盯动词），"吃"同时盯"猫"和"鱼"（动词分别盯主宾）

**假设头1的注意力权重（学会盯"动词→宾语"方向）：**

| | 猫 | 吃 | 鱼 |
|:---:|::|::|::|
| **猫** | 0.40 | 0.35 | 0.25 |
| **吃** | 0.30 | 0.20 | **0.50** |
| **鱼** | 0.30 | 0.30 | 0.40 |

→ 头1学会了"吃"盯"鱼"（动词盯宾语）

### Step 4：加权求和 V

**头0**——用头0的注意力权重对 V 加权平均：

```
V(猫) = [0.34, 0.37], V(吃) = [0.40, 0.32], V(鱼) = [0.36, 0.43]

"猫"的新向量（头0视角）：
  0.15×[0.34, 0.37] + 0.60×[0.40, 0.32] + 0.25×[0.36, 0.43]
  = [0.051+0.240+0.090, 0.056+0.192+0.108]
  = [0.381, 0.356]
  ← "猫"从"吃"那里获取了大量信息（权重0.60）
```

**头1**——用头1的注意力权重对 V 加权平均：

```
"猫"的新向量（头1视角）：
  0.40×[0.34, 0.37] + 0.35×[0.40, 0.32] + 0.25×[0.36, 0.43]
  = [0.136+0.140+0.090, 0.148+0.112+0.108]
  = [0.366, 0.368]
  ← 头1更平均地看"猫"自己，因为它是名词（主语），不需要太关注宾语
```

### Step 5：合并头

```
"猫"最终输出（4维）：
  头0的视角：[0.381, 0.356]
  头1的视角：[0.366, 0.368]
  拼起来：[0.381, 0.356, 0.366, 0.368]

→ 经过 o_proj 混合后，传递给下一层
```

### 这个例子的关键发现

```
头0的全书：    45% 看"吃"
              37% 看"鱼"     ← 单词"吃"关注了几乎所有的 tokens
              18% 看"猫"

头1的全书：    吃→鱼占50%
              猫→自己占40%    ← 名词关注自己较多
              鱼→鱼占40%
```

不同的头自然分化出了关注模式。虽然这里只演示了 2 个头的简单差异，真实 8 个头 + 8 层的组合会产生极其丰富的注意力模式——**每层 8 个方向，8 层层层递进，最终模型既能抓住局部语法，又能理解全局语义** (´▽`ʃ♡ƪ)
