# 🧵 从文本到输出——LLM 六步完整流程

> Fairy 复查后的修正版，2026-07-10

---

## 概览：一条线串起来

```
"我 爱 你"
    │
    ▼
① Tokenizer → [token_id₀, token_id₁, token_id₂]
    │
    ▼
② Embedding → [vec₀, vec₁, vec₂]  （每个 token 查表拿向量）
    │
    ▼
③ 位置编码（RoPE） → 旋转 Q/K，带入位置信息
    │
    ▼
④ QKV 投影 → Q₀..₂, K₀..₂, V₀..₂  （每个 token 有三个身份）
    │
    ▼
⑤ Q × K^T → 注意力分数矩阵 [3×3]
    │
    ▼
⑥ softmax(分数) @ V → 新向量 [3×d]  （每个 token 融合上下文后更新）
```

---

## 第一步：Tokenizer（分词器）

### 训练阶段
```
输入：大量文本 + 指定词表大小（如 6400）
过程：BPE 算法
  初始：每个字/byte 都是独立 token
  循环：找到最常一起出现的相邻 token 对 → 合并成新 token
  直到：词表凑够了 6400 个 token
输出：一串「合并规则」（按频率从高到低排好）
```

### 推理阶段（分词时）
```
输入："我吃了"
过程：
  拆成 byte → [我, 吃, 了]
  按顺序应用合并规则：
    规则 #1: "吃"+"了" → "吃了"  ← 匹配！应用
    规则 #3: "我"+"吃了" → "我吃了"  ← 匹配！应用
  最终：["我吃了"]
输出：token id 序列 [42, 17]（假设 "我吃了"=42, "？"=17）
```

### ❌ 容易混淆的点
- **不是**「最长前缀匹配」——BPE 是按**频率顺序**依次检查能否合并相邻 pair
- 最长前缀匹配是 Unigram Model 的做法，GPT-2/LLaMA 用的是 BPE

### 关键设计
| 特性 | 说明 |
|:----|:-----|
| Byte-level BPE | 先转成 byte（256 个基础），再在上面做 merge，保证任意文本都能分词 |
| 词表大小权衡 | 太小→序列长（费算力）；太大→Embedding 层占参数 |
| 特殊 token | `<|endoftext|>`、`<think>`、`<tool_call>` 等 |

---

## 第二步：Token Embedding（词嵌入）

```
token id 序列：[42, 17, 88]

查表——Embedding 矩阵（6400行，每行768个数）：
第42行 → [0.23, 0.56, -0.12, ..., 0.89]  ← "我吃了"的向量
第17行 → [0.11, -0.34, 0.78, ..., 0.02]  ← "吗"的向量
第88行 → [0.45, 0.03, 0.67, ..., -0.21]  ← "？"的向量

输出：[3 × 768] 的矩阵
```

- Embedding 矩阵是**可训练的**——训练中会不断调整每个向量
- 相似语义的 token，向量会越来越靠近（余弦相似度大）

## 第三步：位置编码

### 绝对位置编码（GPT-2 老方案）

```
在每个 token 的输入向量上「加」一个位置向量：

token向量₀ + 位置向量₀ → 输入模型
token向量₁ + 位置向量₁ → 输入模型
token向量₂ + 位置向量₂ → 输入模型

局限：位置 2049 没学过（没对应的位置向量）→ 用不了
```

### RoPE（旋转位置编码，现代方案）

```
不在输入上加，而是在 Attention 计算前「旋转」Q 和 K：

把 Q 向量分成前后两半：(d₀, d₄), (d₁, d₅), ...
每对在自己的二维平面上旋转「位置 p × 频率」的角度

位置 0：不转
位置 1：转 φ 度
位置 2：转 2φ 度
...

关键：方向变了（位置指纹），长度没变（语义强度保留）
```

| 对比 | 绝对位置编码 | RoPE |
|:----|:-----------|:-----|
| 作用方式 | 加到 token 输入上 | 旋转 Q/K 向量 |
| 位置 2049 | 没学过→报错 | 可扩展到 32k+ |
| 相对位置感知 | 要靠训练学 | 天然有 cos(位置差) |

## 第四步：QKV 投影

每个 token 通过三个不同的矩阵，产生三个不同身份：

```
token 向量（768维）
    │
    ├── × W_Q（768×768）→ Q（768维）  ← "我在找什么"
    ├── × W_K（768×768）→ K（768维）  ← "我有什么被找到"
    └── × W_V（768×768）→ V（768维）  ← "我能提供什么信息"

3个 token → Q₀Q₁Q₂、K₀K₁K₂、V₀V₁V₂（各 [3×768]）
```

**为什么要分开？** Q 和 K 共同决定「注意力花在哪」，V 决定「买了什么货」。K 和 V 需求矛盾（K 要方便被找到，V 要携带丰富语义），分开后各管各的。

**QKV 是临时的**——每层 Attention 前重新算一次，算完就扔。

---

## 第五步：Q × K^T → 注意力分数

```
Q：[3×768]     K^T：[768×3]

Q @ K^T = [3×3]  ← 注意力分数矩阵

       K₀    K₁    K₂
Q₀  [0.75, 0.64, 0.69]
Q₁  [0.58, 0.54, 0.60]
Q₂  [0.66, 0.60, 0.62]

每行含义：「这个位置去看其他所有人的相似度」
每列含义：「这个位置被其他人看的相似度」
```

**然后除以 √d**（防止数字太大），再做 softmax（每行加起来=1）。

---

## 第六步：注意力权重 @ V → 新向量

```
softmax 后的 [3×3]  @  V的 [3×768]  =  [3×768]

取出"我"的行 [0.343, 0.325, 0.332]：
  0.343 × V₀ + 0.325 × V₁ + 0.332 × V₂
  = 新"我"向量（768维）

新向量 = 加权混合了所有 token 的信息
```

**对齐方式**：注意力矩阵的「列数=token 个数」对齐 V 的「行数=token 个数」，所以 @ 能对上。

**@ 的本质**：每个权重 × 对应的 V → 对应位置相加 → 合成新向量。

## 整体流程的维度流转

| 步骤 | 形状变化 | 关键动作 |
|:----|:--------|:--------|
| 输入文本 | 字符串 | — |
| ① Tokenizer | sequence of token IDs | BPE 合并规则 |
| ② Embedding | [seq, d] | 查表 |
| ③ RoPE | 同 [seq, d] | 旋转 Q/K（位置编码） |
| ④ QKV 投影 | [seq, d] → [seq, d]×3 | ×W_Q, ×W_K, ×W_V |
| ⑤ Q×K^T | **[seq, seq]** ← 注意变形状了 | 点积出分数矩阵 |
| ⑥ softmax@V | [seq, seq] @ [seq, d] → **[seq, d]** | 加权求和 |

> d = hidden_size（如 768），seq = token 个数

---

## 知识点关联

| 概念 | 在流程中的位置 | 说明 |
|:----|:------------|:----|
| BPE | ① Tokenizer | 子词分词算法 |
| Embedding 矩阵 | ② 查表 | 6400×768 的可训练矩阵 |
| RoPE | ③ 位置编码 | 旋转 Q/K，不改变长度 |
| QKV 投影 | ④ | W_Q, W_K, W_V 三个独立矩阵 |
| 点积 | ⑤ Q×K^T | 算出相似度分数 |
| Softmax | ⑤→⑥ 中间 | 归一化到 [0,1]，加起来=1 |
| 加权求和 | ⑥ softmax@V | 上下文信息融合 |

---

## 下一步

这 6 步只是**一个 Transformer Block** 的内容。完整的 LLM 有 8 层（minimind）甚至 80 层（大模型），每层都重复这 6 步，但输入越来越「深」，学到越来越抽象的特征。

第三阶段（GPT 完整架构）会补充：
- LayerNorm / RMSNorm——每层之间的归一化
- Feed-Forward Network（FFN）——除了 Attention 外的另一半
- 残差连接——让深层梯度不消失
- 整体拼装——8 层 Block + 输出层
