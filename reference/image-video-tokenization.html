📸 图片与视频 Token 化 — 参考速查

reference 多模态 · 视觉编码

核心思想

图片/视频是连续的像素网格，LLM 只能吃离散的 token 序列。解决方案：把图片切成小方块（patches），每个方块展平后线性投影成一个向量——这样就有了「视觉 token」。

ViT 基本流程（图片 → 视觉 token）

输入图片 (224×224×3)
    │
    ▼
切 patches (16×16) → 14×14 = 196 个 patch
    │
    ▼
每个 patch 展平 (16×16×3 = 768 个像素值)
    │
    ▼
线性投影 (768 → d_model) → 每个 patch 变成一个向量
    │
    ▼
加上位置编码 → 196 个视觉 token
    │
    ▼
送入 Transformer encoder

关键参数

参数常见值含义
Patch Size14, 16, 32每个小方块的边长（像素）
Grid Size14×14, 16×16图片被切成了几行几列
Token 数196, 256Grid 的行 × 列
d_model768, 1024, 1280每个视觉 token 的向量维度

多模态 LLM 的三种接法

🔗 1. 投影层法（LLaVA 方案）

视觉 token 经过一个 MLP 投影层，映射到 LLM 的 token embedding 空间。然后拼在文本 token 前面。

[视觉 token × 196] → [投影层] → [对齐后的 token × 196]
                                          ↓
                     ┌────────────────────┐
                     │  [img_token][img_token]...[sys_prompt][user text][assistant] │
                     └────────────────────┘
                              ↑ 直接拼起来
                     LLM 当它是普通 token 序列处理
优点：简单，改动小。缺点：视觉 token 全是图像底层信息，没有语义抽象。
🔄 2. 交叉注意力法（Flamingo / LLaMA 3.2 Vision）

LLM 每层在 self-attention 之外多一套 cross-attention 层，专门"看"视觉 token。视觉 token 不跟文本 token 拼在一起，而是每层主动去读图片信息。

文本 token: [t₁][t₂][t₃]... → self-attention → cross-attention → FFN
                                                       ↑
                                              视觉 token: [v₁][v₂][v₃]...
优点：视觉和文本各自保持独立空间。缺点：架构改动大，每层多一套参数。
📦 3. Q-Former 压缩法（BLIP-2 / InstructBLIP）

引入一组「学习出来的查询向量（learned queries）」，通过 cross-attention 从视觉 token 中提取信息，压缩成更少的 token。

视觉 token × 256 → [cross-attention] ← 可学习的 32 个查询
                      ↓
             压缩后的视觉 token × 32 → 输入 LLM
优点：可大幅减少视觉 token 数量（256→32）。缺点：多一个模块要训练。

视频 token 化的特殊挑战

方法原理优缺点
帧采样 每隔 N 帧取一张，送 ViT 简单，但丢帧间运动信息
3D 时空 patch 把视频切成小立方体（Time×H×W） 保留时间维度，但参数量×时间轴长度
因果 3D 卷积 (CausalConv3D) 只利用过去帧的信息编码当前帧 适合流式处理，但实现复杂
时空压缩编码器 专用编码器把多帧压缩成少量 token 效率高，但需要额外训练

动态分辨率（Qwen2-VL 方案）

传统 ViT 只能处理固定分辨率。Qwen2-VL 的做法：

  • 把原图按比例缩放到多个候选尺寸
  • 切 patches 时，保持 patch size 不变，但 grid 大小自适应
  • 不同分辨率的图自然产生不同数量的 token
  • 用 M-ROPE（多模态位置编码）区分文本位置和图像位置
  • 结果：一张 1680×2560 的竖屏照片约 2560 个视觉 token；一张 420×640 的缩略图约 160 个

一句话总结

图片 token 化 = 切 patches → 线性投影 → 得到视觉 token 序列。
视频 token 化 = 逐帧取图 → 每帧当图片处理 → 沿时间轴拼起来（但需要机制避免 token 爆炸）。
多模态 LLM = 视觉 token + 文本 token → 同一网络处理。
