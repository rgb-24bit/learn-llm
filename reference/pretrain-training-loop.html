⚙️ Pretrain 训练流程速查

reference 阶段 3 — 训练机制

单步训练流程

┌─────────────────────────────────────────────────────────────┐
│                   一 个 训 练 步 (step)                      │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ① 采样 batch：从数据集中取一批文本                          │
│     → input_ids: [32 × 340]（32 段文本，每段 340 个 token）  │
│     → labels: [32 × 340]（右移一位的 input_ids）             │
│                                                             │
│  ② 前向传播：input_ids → 模型 → logits                      │
│     Embedding → 8×Block → lm_head → logits [32×340×6400]    │
│                                                             │
│  ③ 算 loss：logits vs labels → 交叉熵                       │
│     logits[:, :-1, :] 预测 logits[:, 1:] 的真实 token       │
│                                                             │
│  ④ loss / accumulation_steps → backward → 梯度累积          │
│     每步都反向传播，但只把梯度累加，不更新参数               │
│                                                             │
│  ⑤ 攒够 accumulation_steps 步：                             │
│     a) 梯度裁剪（最大值限制在 1.0）                          │
│     b) optimizer.step() — AdamW 用梯度更新参数              │
│     c) optimizer.zero_grad() — 清空梯度                     │
│     d) 更新学习率（余弦衰减）                                │
│                                                             │
│  ⑥ 每隔 N 步：打印 loss + 保存 checkpoint                   │
│                                                             │
└─────────────────────────────────────────────────────────────┘

Loss 计算细节

input_ids:  [token₁][token₂][token₃][token₄]...[token₃₄₀]
labels:          [token₂][token₃][token₄][token₅]...[PAD]

模型预测：token₁ 的位置 → 预测 token₂ 的概率分布
          token₂ 的位置 → 预测 token₃ 的概率分布
          token₃ 的位置 → 预测 token₄ 的概率分布
          ...

每个位置输出一个 vocab_size 维的概率向量（6400 个值）
loss 衡量："模型认为 token₂ 的概率" vs "实际就是 token₂"

关键超参数速查

参数minimind 值工业级（LLaMA 等）什么意思
batch_size 32 1M~4M token 每次喂多少样本
accumulation_steps 8 1~8 攒几步梯度再更新
有效 batch 32×8=256 数万 实际用于更新的样本量
max_seq_len 340 4096~128K 每段文本截断长度
learning_rate 5e-4 1e-4~3e-4 初始学习率
lr 衰减 余弦 → 10% 终值 余弦 → 10% 终值 学习率随训练逐渐降低
grad_clip 1.0 0.5~1.0 梯度最大值上限
optimizer AdamW AdamW 行业标准
dtype bfloat16 bfloat16 混合精度训练
epochs 2 1~3 过几遍数据
训练 token 总数 ~2.1B 1T~15T 看了多少文本

余弦学习率衰减

公式：lr = lr_max × (0.1 + 0.45 × (1 + cos(π × step / total_steps)))

直观形状：
lr
↑
│   ▁▁▁▁▁▁▁▁▁▁▁
│  ▕              ▔▔▔▔▔▔▔▔▔▔▔▔
│ ▕                          ▔▔▔
│▕                               ▔▔▔
└─────────────────────────────────────→ 训练步数

初始 lr = 5e-4 × (0.1 + 0.45×2) = 5e-4 × 1.0 = 5e-4
最终 lr = 5e-4 × (0.1 + 0.45×0) = 5e-4 × 0.1 = 5e-5

训练 vs 推理的差异

维度训练推理
输入方式整段文本并行输入逐 token 自回归
KV Cache不用（每次都算整段）用（缓存历史 K/V）
需要梯度✅ 要（反向传播更新参数）❌ 不要（torch.inference_mode）
Loss❌ 需要计算❌ 不需要
Dropout✅ 启用❌ 关闭
数据类型bfloat16 + fp32 master weightsbfloat16/fp16/int8/4bit

一句话总结

训练循环 = 取 batch → 前向算 loss → 反向算梯度 → 攒够步数 → 裁剪梯度 → AdamW 更新参数 → 清梯度 → 重复。

每一个 step 更新一次参数，几万个 step 后模型逐渐学会"下一个词预测"。
全过程不需要人类标注——互联网上的文本就是天然标签。
