CLIP 对比学习机制

训练数据：N 对图文对（图片₁, 文本₁）、（图片₂, 文本₂）…（图片ₙ, 文本ₙ）

目标：让配对项在 embedding 空间靠近，非配对项远离

   图片₁──→[图片编码器]──→ v₁ ──┐
   图片₂──→[图片编码器]──→ v₂ ──┤         相似度矩阵 (N×N)
   图片₃──→[图片编码器]──→ v₃ ──┤    ┌─────────────────────┐
   ...                         ├───→│ ✓ 高  ✗ 低  ✗ 低  ✗ │
   图片ₙ──→[图片编码器]──→ vₙ ──┤    │ ✗  ✓ 高  ✗ 低  ✗  │
                                 │    │ ✗  ✗  ✓ 高  ✗  │
   文本₁──→[文本编码器]──→ t₁ ──┤    │ ✗  ✗  ✗  ✓ 高  │
   文本₂──→[文本编码器]──→ t₂ ──┤    └─────────────────────┘
   文本₃──→[文本编码器]──→ t₃ ──┤        对角线 = 配对
   ...                         │         非对角线 = 不配对
   文本ₙ──→[文本编码器]──→ tₙ ──┘
InfoNCE Loss 伪直觉

对每个图片向量 vᵢ：在所有文本向量 t₁~tₙ 中，找出哪个才是它配对的 tᵢ。
模型要做的事：v₁ 和 t₁ 的相似度，要比 v₁ 和 t₂、t₃…tₙ 的相似度都高。
最终变成一个 N 选 1 的分类问题——比对的越准，loss 越低。

多模态 LLM 训练流水线

阶段训练数据训练哪些参数目的
0. CLIP 预训练 4 亿图文对 视觉编码器 + 文本编码器 建立图文对齐的 embedding 空间
1. 连接器对齐 图文对（同 CLIP 数据） 🟢 连接器
🔴 冻结 ViT + 冻结 LLM		 让视觉 token 能"翻译"进 LLM 的空间
2. 多模态 SFT 图片问答数据 🟢 连接器 + LLM（LoRA）
🔴 冻结 ViT		 教会 LLM 看懂图，回答问题
3. 多模态对齐 图片偏好数据 🟢 LLM（LoRA 或全量） 提高回答质量（可选阶段）

一句话总结

图文如何关联？ — CLIP 对比学习：拉近配对图文，推远非配对图文。
多模态模型怎么训练？ — 三步走：CLIP 预训练 → 对齐连接器 → 多模态 SFT。
整体是什么结构？ — 视觉编码器 + 连接器 + LLM 主干的三明治架构。
