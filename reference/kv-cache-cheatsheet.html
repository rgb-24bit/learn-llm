🗂️ KV Cache 速查

核心公式

无 Cache: KV 总计算量 ∝ n(n+1)/2 ≈ O(n²) 有 Cache: KV 总计算量 ∝ n ≈ O(n) n 为生成 token 总数 每 token 新增 cache = 2 × num_layers × num_heads × head_dim × dtype_bytes

每步做什么

阶段操作QKV
Prefill整条 prompt 一次算完全算（用完扔）全算→存全算→存
Decode 第 t 步生成第 t+1 个词只算新的（用完扔）只算新的→追加只算新的→追加
AttentionQ_new × [K_cache, K_new]—从 cache 取全部从 cache 取全部

内存估算公式

KV_Cache_size = 2 × num_layers × num_heads × head_dim × seq_len × dtype_bytes

举例：LLaMA 7B, 4K seq, float16
  = 2 × 32 × 32 × 128 × 4096 × 2
  = 2 × 32 × 32 × 128 × 4096 × 2
  = 2,147,483,648 bytes
  = 2 GB

相关术语

术语定义
Prefill首次 token 生成：整条 prompt 一次算完，填满 KV Cache
Decode逐 token 生成：每步只算 1 个新 token，追加 Cache
Streaming LLM只保留最近 N 个 token 的 KV Cache，丢弃中间历史
PageAttentionvLLM 的 KV Cache 管理：像虚拟内存一样分页管理，减少碎片
KV 量化把 Cache 里的 float 压缩成 int8/int4，省显存
