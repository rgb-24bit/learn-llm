🏗️ MoE 速查

一句话

把 FFN 拆成多个专家，每步只激活 Top-K 个。 差异只发生在 FFN，Attention 不变，KV Cache 不变。

架构对比

DenseMoE
FFN1 个大的N 个小的（专家）
激活全部参数Top-K 专家
每层独不独立—每层有一套独立专家
Router—Linear(hidden, num_experts) → softmax → Top-K
专家分工—训练中自然涌现，非人为预设
KV Cache有同 Dense（不受影响）

Router 计算流程

输入 x (d维)
  → x @ W_router (d × num_experts) → 原始分数
  → softmax → 概率分布
  → topk(k=num_experts_per_tok) → 选中专家编号 + 权重
  → 归一化权重 → 分发给对应专家 → 加权合并输出

关键术语

术语含义
Router/Gate决定 token 去哪个专家的线性层
Expert一个独立的小 FFN（gate/up/down 三层）
Top-K每 token 激活 K 个得分最高的专家
Aux Loss / Load Balance Loss防止 Router 垄断的惩罚项
Shared ExpertDeepSeek 方案：每个 token 必经的通用专家
Fine-Grained ExpertDeepSeek 方案：拆得更细的专家（64~160 个）
