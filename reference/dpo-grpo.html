⚡ DPO & GRPO 速查表

reference 阶段 4 — 偏好对齐

一、DPO 核心公式

loss = -logsigmoid( β × [ (chosen_policy - reject_policy) - (chosen_ref - reject_ref) ] )

其中：
  chosen_policy  = Σ log P_policy(chosen_token | context)    策略模型给好回答的 log 概率
  reject_policy  = Σ log P_policy(reject_token | context)    策略模型给差回答的 log 概率
  chosen_ref     = Σ log P_ref(chosen_token | context)       参考模型给好回答的 log 概率
  reject_ref     = Σ log P_ref(reject_token | context)       参考模型给差回答的 log 概率
  β              = 偏离惩罚系数（minimind 默认 0.15）

关键变量：β 越大 = 越紧贴参考模型
          β 越小 = 越可以自由偏好调整

DPO 损失函数通俗版

DPO的"隐式奖励" = (策略模型偏好的程度) - (参考模型本来就偏好的程度)
loss  = -logsigmoid(β × 隐式奖励)

当隐式奖励 > 0 → loss 小 → 策略已经学会偏好了 ✓
当隐式奖励 < 0 → loss 大 → 策略还不如参考模型，需要继续学 ✗

二、GRPO 核心公式

GRPO 的关键创新：没用 Critic 模型算 baseline，而是用组内标准化

Step 1: 组内标准化（代替传统 Critic）
  advantages = (rewards - mean(rewards_in_group)) / std(rewards_in_group)

Step 2: PPO-clip 目标（和 PPO 一样）
  ratio = exp(policy_logprob - old_policy_logprob)
  loss1 = ratio × advantage              (直接优化)
  loss2 = clip(ratio, 1-ε, 1+ε) × advantage  (裁剪版)
  policy_loss = -min(loss1, loss2)

Step 3: KL 惩罚（防止跑偏）
  per_token_kl = exp(ref - policy) - (ref - policy) - 1
  不对称：概率变高惩罚 > 概率变低惩罚（鼓励谨慎）

Step 4: 总 loss
  total_loss = policy_loss + β × KL_penalty

四、GRPO 奖励设计（minimind 示例）

条件分值
回答长度 20~800 字+0.5
回答长度超出范围-0.5
包含 <think> 且思考部分 20~300 字+1.0
包含 <think> 但思考部分超范围-0.5
恰好 1 个 </think> 标签+0.25
重复惩罚最高 -0.5
奖励模型打分任意（可正可负）
