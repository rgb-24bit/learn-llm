📋 LLM 基准速查表

reference 阶段 5 — 评估与数据

一、知识类基准

MMLU — Massive Multitask Language Understanding

属性值
发布年份2020
论文arXiv:2009.03300
题目类型4 选 1 选择题
总题数15,908
学科数57（STEM/人文/社科/其他）
评分方式准确率（字母匹配或 log-prob）
默认 shot5-shot
随机基线25%
人类水平~89%
前沿（2026）89–93%
陷阱选择题利于排除法；前沿饱和中
后继MMLU-Pro（10 选 1）

HellaSwag — 常识推理

属性值
发布2019
格式4 选 1（选最合理的情节结局）
题数~10,000
人类水平~95%
前沿95%+（已饱和）

TruthfulQA — 抵抗误解

属性值
发布2021
格式开放回答（是否传播了常见错误观念）
题数817
意义测的是对齐质量，不是知识量
前沿~70%（仍未饱和）

二、推理类基准

GSM8K — 小学数学

属性值
发布2021 (OpenAI)
题数8,500
难度2–8 步算术
前沿99% ✅ 已饱和
核心贡献推动了 Chain-of-Thought 普及
污染影响去掉重复题后准确率降 ~13%

MATH — 竞赛数学

属性值
发布2021
题数12,500
难度AMC / AIME 竞赛级
前沿~85%（接近饱和）

GPQA — 研究生级问答

属性值
发布2023
格式4 选 1
难度专家级（物理/生物/化学）
前沿~94%（Gemini 3.1）
特点即使专家也只有 ~65% 的准确率

三、代码类基准

HumanEval

属性值
发布2021 (OpenAI)
题数164
任务根据 docstring 写 Python 函数
评分pass@k（通过单元测试）
前沿90%+ ✅ 已饱和
不足太独立，不反映真实工程

SWE-bench

属性值
发布2024
题数2,294
任务修真实 GitHub Issue
评分生成补丁通过测试
前沿~80%（SWE-bench Verified）
特点至今无模型满分，当前最难代码基准

四、综合/偏好类

Chatbot Arena (LMSYS)

属性值
运营方LMSYS（UC Berkeley）
地址lmarena.ai
方法匿名对战 → 人类投票 → Elo 评分
评分Elo（基准 ~1000，前沿 ~1450）
优点最接近真实使用体验
缺点受人口偏差影响（投票者偏技术背景）

MT-Bench

属性值
发布2023
题数80 道多轮对话
评分LLM-as-Judge（用 GPT-4 打分）
格式8 个类别 × 5 个 prompt × 2 轮对话

五、基准陷阱一览

陷阱说明应对
📈 饱和所有模型接近天花板，失去区分度关注未饱和基准（SWE-bench, GPQA）
💉 数据污染测试题出现在训练数据中使用动态基准如 LiveBench
🎨 格式刷分模型学会格式而非内容交叉验证不同格式的基准
🏭 基准≠生产满分不代表实际好用建立自己的应用级评估集

六、前沿模型最新趋势

截至 2026 年中：

  • 知识型基准（MMLU, HellaSwag）已全面饱和，MMLU-Pro 成为新标准
  • 推理型基准（GPQA, AIME）仍有区分度，是当前战场
  • 代码型基准（SWE-bench）是增长最快的评估方向
  • 动态基准（LiveBench, HLE）正在取代静态基准成为主流
  • Agent 评估（SWE-bench Agent, WebArena）是最新前沿
