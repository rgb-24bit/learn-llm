🧠 图片训练与图文对齐

lesson 0004 阶段 3 — 多模态 · 训练机制

上一课你知道了图片怎么变成 token。但 token 只是材料——模型怎么"学会"看图？图和文字怎么关联？多模态模型整体长什么样？

这一课全部回答。

1. 核心问题：图片和文字是两套语言

如果你只把视觉 token 扔进 LLM，模型会看到一堆数字——但它不知道这些数字对应什么语义。

就像你给一个只懂英语的人一本中文书：他看得见符号，但看不懂意思。

要让模型"看懂"图片，必须先把图片和文字的语义对齐到同一个空间。而完成这个对齐的核心技术叫 CLIP。

2. CLIP：图文对齐的"翻译官"

2.1 训练数据

CLIP 的训练数据很简单粗暴：4 亿张图片，每张配一句文字描述（从网上爬的）。

比如：

  • 一张猫的照片 → "A cat sitting on a chair"
  • 一张汽车的照片 → "A red car parked on the street"
  • 一张蛋糕的照片 → "A birthday cake with candles"

2.2 训练方法：对比学习

CLIP 同时训练两个编码器：

  • 图片编码器（ViT）— 把图片变成向量
  • 文本编码器（Transformer）— 把文字变成向量

拿具体数字走一遍训练过程：

🔢 假设 batch size = 4

你有 4 张图片和对应的 4 句描述：

  • 猫图 ↔ "一只猫"
  • 狗图 ↔ "一只狗"
  • 车图 ↔ "一辆车"
  • 蛋糕图 ↔ "一个蛋糕"

第一步：图片编码器把 4 张图变成 4 个向量 v₁ v₂ v₃ v₄

第二步：文本编码器把 4 句话变成 4 个向量 t₁ t₂ t₃ t₄

第三步：算相似度矩阵（4×4）

         t₁(猫)  t₂(狗)  t₃(车)  t₄(蛋糕)
v₁(猫图)   0.95    0.20    0.15    0.18    ← 猫图跟"猫"最像 ✓
v₂(狗图)   0.22    0.92    0.13    0.16    ← 狗图跟"狗"最像 ✓
v₃(车图)   0.10    0.12    0.93    0.08    ← 车图跟"车"最像 ✓
v₄(蛋糕图) 0.15    0.14    0.11    0.94    ← 蛋糕图跟"蛋糕"最像 ✓

第四步：计算 loss

模型要做的：对图片 v₁，"找"出正确的文本 t₁，且 v₁·t₁ 的相似度 > v₁·t₂、v₁·t₃、v₁·t₄。

这本质上是一个 4 选 1 的分类任务。模型把 v₁ 在 4 个文本中选出正确的那一个，就算做对。

如果 v₁ 选了 t₂（狗）并给出高分——loss 就大，模型被惩罚。

2.3 InfoNCE Loss（对比损失）

名字唬人，本质就是一个 softmax 分类：

对图片 vᵢ：在 N 个文本中，让正确配对 tᵢ 的相似度得分经过 softmax 之后接近 1，其他接近 0。

如果你用 CrossEntropyLoss 做过分类任务，这里一样的——只不过"类别"是 N 个文本。

训练结束后，CLIP 得到了什么？
两个编码器学会了同一个 embedding 空间——"猫"的文字向量和猫的图片向量之间的距离很近，"猫"和"汽车"的图片向量距离很远。

这个共享语义空间，就是后续多模态 LLM 的基石。

3. 多模态 LLM 的整体结构

有了 CLIP 对齐好的视觉编码器，多模态 LLM 就是"三件套"拼在一起：

4. 多模态模型的训练过程（三步走）

你已经知道纯文本 LLM 是 Pretrain → SFT → DPO。多模态 LLM 类似，但要再加一步专门对齐视觉和语言：

第 0 步：CLIP 预训练（图文对齐的基础）

项目值
数据4 亿图文对（网上爬的）
训练谁视觉编码器 + 文本编码器（两套独立的网络）
怎么算 loss对比学习（上面讲过的 InfoNCE）
在哪做OpenAI 等研究机构做完，开源出来供大家用
结果一个能"看懂"图片语义的视觉编码器
关键理解：大多数做多模态 LLM 的团队 不自己训练 CLIP。就像你训练 LLaMA 不需要自己发明 Transformer——直接用现成的 CLIP ViT 模型（OpenAI 开源的）。

第 1 步：对齐连接器（只训投影层）

项目值
数据图文对（和 CLIP 类似的数据）
训练谁只训连接器，冻结 ViT + 冻结 LLM
训练目标LLM 看到视觉 token 后，能预测出正确的文字描述
训练数据量约 60 万图文对
为什么只训连接器？ViT 已经是"看懂图"的、LLM 已经是"能说会道"的。唯一的缺口是连接器——它要学会把 ViT 的输出"翻译"给 LLM

第 2 步：多模态 SFT（教会模型看图回答问题）

项目值
数据图片 + 问题 + 答案（人工标注的问答对）
训练谁连接器 + LLM（LoRA），仍冻结 ViT
典型数据"这张图里有什么？" → "有一只橘猫" / "这张图的构图如何？" → "三分法构图……"
数据量LLaVA 用了约 15 万条
关键洞察：LLaVA 的 15 万条多模态 SFT 数据不是人工从头写的——而是用 GPT-4V（已有的最强多模态模型）看图生成问答对。先用最强模型造数据，再用这些数据训练更小的模型。这叫 self-instruct 的多模态版本。

5. 具体例子：模型怎么"看懂"一张图

走一遍完整的前向推理：

场景：你上传了一张猫的照片，问"这是什么？"
  1. 图片进入 ViT：224×224 照片 → 切 196 个 patches → 线性投影 → 196 个视觉 token（每个 768 维）
  2. 视觉 token 过连接器：MLP 投影层把 768 维映射到 LLM 的 4096 维空间
  3. 拼成完整序列：[视觉 token × 196] [系统指令] [用户问题："这是什么？"]
  4. LLM 处理：Self-Attention 让每个视觉 token 能看到其他视觉 token，也能看到文本 token——模型在"读图的同时看问题"
  5. 生成回答：逐 token 生成 "这" → "是" → "一" → "只" → "猫"
Attention 机制在此时发挥作用：生成"猫"这个 token 时，模型会重点关注图片中猫所在区域对应的几个 patch token。

6. 纯文本 LLM 训练 vs 多模态训练对比

纯文本 LLM多模态 LLM（额外部分）
预训练 TB 级文本，"下一个词预测" CLIP：4 亿图文对，对比学习
对齐层 无（文本本来就在词表里） 连接器对齐：60 万图文对，只训投影层
SFT 纯文本对话数据 图片问答数据（15 万条）
DPO/RLHF 文本偏好数据 图片偏好数据（可选）
总训练成本 极高的百万元级 在一个已经训练好的 LLM 上加，可能几千元（LlAVA）

7. 想一下

🤔 思考一：CLIP 和 GPT 的预训练有什么本质区别？

GPT 预训练的任务是"下一个词预测"——在文本内部学习。

CLIP 的任务是"这张图和这句话是不是一对"——在两种模态之间学习。

一个学语言本身，一个学图文关联。它们是互补的。

🤔 思考二：为什么连接器对齐阶段要冻结 LLM？

因为 LLM 已经学会了完美的语言能力。如果一起训，连接器可能会"带偏"LLM——比如视觉 token 噪声太大，LLM 为了迁就噪声而降低自己的语言能力。

先固定 LLM，强迫连接器去适应 LLM；连接器稳定了，再放开一起训。
