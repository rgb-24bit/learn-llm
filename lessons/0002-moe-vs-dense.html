🏗️ MoE vs Dense——混合专家模型架构

lesson 0002 阶段 3 — GPT 架构 · MoE

MoE（Mixture of Experts）是当前 LLM 架构的最大分水岭之一——Dense 模型每步激活所有参数，MoE 每步只激活一部分「专家」。两者差异 只发生在 FFN 层，Attention 部分完全一致。

1. 核心差异：只改 FFN

Dense Block: 输入 → Pre-Norm → Attention → +残差 → Pre-Norm → [ FFN ] → +残差 ↑ 全部参数激活 MoE Block: 输入 → Pre-Norm → Attention → +残差 → Pre-Norm → [ Router → Top-K 专家 ] → +残差 ↑ 只激活 2/8 的参数

为什么只能改 FFN？

AttentionFFN
依赖其他位置？依赖（需全部历史的 K/V）不依赖（只看自己）
拆专家可行？不可行可行
作用收集邻居信息（开会）独立加工信息（消化）

2. MoE 核心机制

Router（路由器）

一个简单线性层：W_router: hidden_size → num_experts。输入 token 向量，输出每个专家的得分，softmax 转概率后取 Top-K。

Router 流程： 输入 x (768维) → x × W_router (768×8) → 8 个 logit → softmax → [0.02, 0.78, 0.01, ...] ↑ 选 Top-2

专家分工是自然涌现的

  • 初始化时所有专家参数随机，无预设角色
  • 训练中相似的 token 倾向于走相同专家 → 该专家参数朝该方向调整
  • 训练完成 → 专家自然分化：语法专家、科学专家、情感专家……
  • 角色名称是人类事后贴的标签，不是人预设的

每层独立

Block 0 的 8 个专家和 Block 1 的 8 个专家是完全独立的不同参数。32 层 × 8 专家 = 256 个专家。浅层专家处理语法/短语，深层专家处理语义/推理。

4. Load Balancing Loss（负载均衡损失）

如果不加控制，Router 会偷懒——所有 token 密集选 2~3 个专家，其他闲置。解法是在 loss 中加入惩罚项：

aux_loss = (每个专家被选中的比例 × 每个专家的平均 Router 概率).sum()
            × num_experts × coef

某个专家被滥用 → 选中比例高 → loss 变大 → Router 被惩罚 → 下次分给其他专家。
