⚙️ Pretrain 训练循环

lesson 0005 阶段 3 — 训练机制 · 核心流程

你已经知道了 GPT Block 的结构、Attention 的原理、Loss 怎么算。但 —— 所有这些是怎么凑在一起跑起来的？

这一课带你完整走一遍一个训练步：从数据进来到参数更新。

0. 先想想全局

你之前学过了 LLM 训练三阶段：Pretrain → SFT → DPO。这一课只说 Pretrain。

Pretrain 的目标极其简单："给我看一堆文本，学会预测下一个词"。

关键理解：Pretrain 不需要任何人工标注数据。互联网上的所有文本都是天然训练材料——每句话的"下一个词"就是标签。这是 LLM 能规模化的根本原因。

好的，现在我们用一个具体模型（minimind 64M）走一遍训练过程。

1. 数据准备：文本 → 数字

原始数据是纯文本，比如维基百科的一段文章。第一步：

  • Tokenizer（BPE）把文本切成 token ID 序列
  • 每段截断或填充到固定长度（minimind 用 340 个 token）
  • 最后一批数据长这样：[32 × 340] —— 一次取 32 段文本，每段 340 个数字

训练时，输入就是 input_ids，标签就是右移一位的 input_ids：

文本原文："今天天气真好"（假设 token = [101, 102, 103, 104, 105]）

input_ids:  [101] [102] [103] [104] [105]
labels:     [102] [103] [104] [105] [PAD]
              ↑     ↑     ↑     ↑
              模型任务：每个位置预测下一个 token

模型在位置 1 看到 [101]，要预测 [102] 是"天气"；位置 2 看到 [102]，要预测 [103] 是"真好"…

2. 前向传播：数据流过模型

拿到一个 batch 后，一次前向传播走到底：

input_ids → [32 × 340]
    │
    ▼
Token Embedding → [32 × 340 × 768]
（每个 token ID 查表，变成 768 维向量）
    │
    ▼
+ Position Embedding (RoPE, 在 Attention 里旋转)
    │
    ▼
× 8 层 Transformer Block (Pre-Norm → Attention + 残差 → FFN + 残差)
    │
    ▼
RMSNorm → [32 × 340 × 768]
    │
    ▼
lm_head（线性层 768 → 6400）→ [32 × 340 × 6400]
                                   ↑      ↑      ↑
                                batch  位置   词表概率

最终输出 logits：一个巨大的 3 维数组。32 段 × 340 个位置 × 6400 个词表项。

每个位置的"6400 个值"，是模型对这个位置"下一个 token 应该是什么"的预测——哪个索引的数值越大，模型就越倾向于预测那个 token。

3. 算 Loss

现在模型输出了 logits（预测），你手上又有 labels（真实答案）——比一比差距有多大。

一个具体位置的例子

假设词表只有 5 个词 [猫, 狗, 吃, 鱼, 跑]。

位置真实下一个 token模型预测（概率分布）loss 贡献
"猫"的位置 "吃"（索引 2） 猫 0.1 / 狗 0.1 / 吃 0.6 / 鱼 0.1 / 跑 0.1 -ln(0.6) ≈ 0.51 ✓
"吃"的位置 "鱼"（索引 3） 猫 0.4 / 狗 0.4 / 吃 0.05 / 鱼 0.1 / 跑 0.05 -ln(0.1) ≈ 2.3 ✗

模型在"吃"位置预测错了——它觉得大概率是"猫"或"狗"（各 0.4），但正确答案是"鱼"（只有 0.1），所以 loss 高达 2.3。模型会受到大的惩罚。

一个 batch 里所有位置加起来取平均，就是最终的 loss 值。

直观理解：loss 相当于"模型有多困惑"。loss=2.3 相当于模型在 10 个选项里蒙（e²·³≈10），loss=0.1 相当于模型有 90% 以上的把握猜对。训练就是从"蒙"到"准"的过程。

4. 反向传播：让模型"知道"自己错了

Loss 是一个数字——比如 3.5。模型怎么利用这个数字来改进？

反向传播（backpropagation）：从 loss 出发，沿着网络往回走，逐层计算每个参数对 loss 的"贡献"有多大。

用之前学过的知识分解：

顺序：loss → cross_entropy → lm_head → 最后一层 FFN → 最后一层 Attention
       → 倒数第二层 FFN → 倒数第二层 Attention → ... → Embedding

每个参数 W 得到一个梯度 g = ∂loss/∂W
梯度的含义：W 增加一丢丢，loss 会变化多少
梯度为正 → 参数往小了调（参数过大导致 loss 高）
梯度为负 → 参数往大了调（参数过小导致 loss 高）

8 层 transformer 全部走一遍，每个参数都有一个梯度值。

注意：训练时 不用 KV Cache。训练是整段文本并行计算的，不存在"重复计算历史位置"的问题。KV Cache 只在推理时用。

5. 梯度累积和更新

现在有了梯度。但直接更新参数有个问题：

batch_size=32，这个数字太小了。用 32 个样本算出的梯度"噪声"很大，更新方向可能不准。但显存有限，一次放不下更大的 batch。

解决方案：梯度累积

step 1: forward + backward → 梯度 g₁（缓存，不更新）
step 2: forward + backward → 梯度 g₂（累加到 g₁）
step 3: forward + backward → 梯度 g₃（累加到 g₁+g₂）
...
step 8: forward + backward → 梯度 g₈（累加到前 7 步）
        ↓
        总梯度 = (g₁+g₂+...+g₈) / 8 ← 等效于 batch=32×8=256
        裁剪梯度（超过 1.0 的截断到 1.0）
        optimizer.step() → 更新参数
        optimizer.zero_grad() → 清空梯度缓存
        重新从 step 1 开始

用 8 步攒出的梯度更新一次——效果等价于 batch=256，但每步显存只用装 32 条数据。

6. 学习率衰减

训练开始时，参数离最优值很远，用大的学习率（5e-4）快步走。训练快结束时，参数已经很接近最优了，用小步慢走防止"跨过头"。

minimind 用的余弦衰减：

初始学习率：5e-4（训练开始）
最终学习率：5e-5（最高值的 10%）

学习率变化类似一条余弦曲线——前半段基本持平，后半段逐渐下降。
训练进度学习率行为
0%5e-4大步探索
25%~5e-4基本不变
50%~4.5e-4开始缓降
75%~2.8e-4明显下降
100%5e-5精细微调

7. 整个训练过程在时间上的样子

minimind 64M 的完整训练：

数据集：~2.1B token（约 600 万段文本，每段 340 token）
总步数：约 24,000 step / epoch × 2 epochs = 48,000 步
训练时间：约 1-2 天（单张 RTX 3090）

loss 变化曲线：
Epoch 1:
  step 0:     loss ≈ 7.0  →  "对词表完全随机猜，ln(6400)≈8.8，还没开始学"
  step 1,000: loss ≈ 5.0  →  "学了一些高频词：了、的、是"
  step 5,000: loss ≈ 3.5  →  "基本语法学会了"
  step 12,000:loss ≈ 2.5  →  "常见搭配掌握了"
  step 24,000:loss ≈ 2.0  →  "第一遍过完"

Epoch 2:
  step 24,001:loss ≈ 2.0  →  "第二遍，同一批数据再看一次"
  step 36,000:loss ≈ 1.5  →  "更精确了"
  step 48,000:loss ≈ 1.2  →  "训练结束"
思考一下：loss 从 7.0 降到 1.2，意味着模型对每个 token 的平均预测置信度从 0.09%（1/6400）提升到了 30%（e⁻¹·²≈0.3）。模型确实"学到"了语言规律，而不是死记硬背。

8. 回头看：你已经在脑子里走了一遍训练循环

🔁 一个训练步的完整流程
  1. 取数据：从数据集拿一个 batch（32 × 340 token IDs）
  2. 前向：Embedding → 8×Block → lm_head → logits [32×340×6400]
  3. 算 loss：交叉熵（预测 vs 真实下一个词）
  4. 反向：从 loss 往回算每个参数的梯度
  5. 累积：把梯度加到缓存里（每步都做）
  6. 攒够 8 步后：裁剪梯度 → AdamW 更新参数 → 清梯度
  7. 调学习率：按余弦衰减降低学习率
  8. 打日志：每隔 100 步打印 loss，每隔 1000 步保存 checkpoint
  9. 重复：走 48,000 步——模型就训练好了
