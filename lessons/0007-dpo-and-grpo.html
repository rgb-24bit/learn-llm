🎯 DPO 与 GRPO

lesson 0007 阶段 4 — 后训练 · 对齐

SFT 让模型学会了"问问题要回答"，但它解决不了一个根本问题——没有人告诉模型什么样的回答更好。

Pretrain 学的是"可能"的 token。SFT 学的是"应该"的 token。DPO/GRPO 学的是"更好"的 token。

关键区别在于：SFT 只能学"正确的"回答，DPO/RLHF 可以学"更好的"回答——哪怕标签数据里没有完美答案，模型也能从对比中学会辨别好坏。

2. DPO — 直接偏好优化

2.1 为什么需要 DPO？

传统的 RLHF（基于强化学习的人类反馈）需要四件套：策略模型 + 参考模型 + 奖励模型 + PPO 算法。训练过程像马戏团杂耍——一边让策略模型生成回答，一边让奖励模型打分，然后用 PPO 更新策略，同时 KL 惩罚防止跑偏。复杂、不稳定、显存爆炸。

2023 年 Stanford 提出的 DPO（Direct Preference Optimization） 发现了一个惊人的事实：奖励模型根本不需要——偏好对比本身就隐含了奖励信号。

核心洞察：RLHF 的奖励模型本质上是"给回答打分"，而偏好数据（回答 A > 回答 B）本身就是一种打分。DPO 直接把这个"隐含的打分"翻译成 loss，省掉奖励模型这个中间商。

2.2 DPO 的工作流

准备：一个 prompt，两份回答（chosen ✓ / rejected ✗） ↓ Step 1：用当前策略模型计算两份回答的 log 概率 ↳ chosen P = 1.2（正确回答的累积概率） ↳ rejected P = 0.4（错误回答的累积概率，也低！） Step 2：用初始参考模型（冻结）也算一遍 Step 3：DPO loss = -logsigmoid( β × [(chosen - rejected)策略 - (chosen - rejected)参考] ) ↓ 优化目标：让策略模型 更偏爱好回答 > 参考模型原本就偏好的程度

2.3 具体数字算例

一个 Batch 的完整计算

原始数据：

变量含义数值
chosen_ref_logprob参考模型给好回答的概率-2.5
reject_ref_logprob参考模型给差回答的概率-3.8
chosen_policy_logprob策略模型给好回答的概率-2.0
reject_policy_logprob策略模型给差回答的概率-5.0
chosen_pref= -2.0 - (-5.0) = 3.0策略明显偏好 chosen
ref_pref= -2.5 - (-3.8) = 1.3参考模型也偏好，但幅度小
implicit_reward= 3.0 - 1.3 = 1.7策略比参考更偏好 chosen
logits= β × 1.7（β=0.15）= 0.255
loss= -logsigmoid(0.255) = -ln(0.563)≈ 0.574 ✓ 已经比较小了

如果策略模型没学会（chosen 和 rejected 的概率差不多）：

chosen_ref = -2.5,  reject_ref = -3.8  →  ref_pref = 1.3
chosen_pol = -2.8,  reject_pol = -3.5  →  pol_pref = 0.7
implicit_reward = 0.7 - 1.3 = -0.6
logits = 0.15 × (-0.6) = -0.09
loss  = -logsigmoid(-0.09) = -ln(0.477) ≈ 0.739  ← 更大的 loss

2.4 β（beta）的作用

β 控制偏离参考模型的程度：

β行为效果
小（如 0.01）惩罚弱，模型可以大胆偏离参考学得快，但也容易忘掉原有知识
大（如 0.5）惩罚强，模型必须紧贴参考稳定，但偏好调整非常缓慢
minimind 默认 0.15中等安全平衡
DPO 的微妙之处：因为不需要奖励模型，DPO 在同一个 prompt 只用一对（chosen/rejected）数据时效果最好。如果一个 prompt 有多个回答要做排序，DPO 就不如传统 RLHF 灵活了。

2.5 DPO 的超低学习率

minimind 的 DPO 学习率 = 4e-8，比 SFT（1e-5）还要再低 250 倍！

阶段学习率相对 Pretrain
Pretrain5e-4基准
SFT1e-51/50
DPO4e-81/12,500

原因很简单：DPO 是在 SFT 已经调好的模型上继续调优，只需要微调概率分布的倾斜方向，动太多反而会遗忘 SFT 学到的对话能力。

3. GRPO — 群体相对策略优化

3.1 从 DPO 到 GRPO

DPO 虽然简单，但有个限制：它需要别人准备好"好回答 vs 差回答"的对比数据。

如果模型能自己生成一堆回答，自己判断哪个好，然后学习——是不是更强大？

这就是 GRPO（Group Relative Policy Optimization），DeepSeek-R1 使用的核心技术。

3.2 GRPO 独有：不需要奖励模型和评论家

传统 RLHF 需要两个额外模型：

  • 奖励模型：给回答打分
  • 评论家（Critic）：估算"这个行动值多少分"（PPO 需要的基础线）

GRPO 把两个都砍掉了：

  • 奖励来自规则 + 小奖励模型（不是大奖励模型）
  • 基础线来自组内统计——对同一个 prompt 生成多个回答，用这组的均值/方差做标准化

3.3 GRPO 完整工作流

┌─────────────────────────────────────────────────────┐ │ 一次 GRPO 训练步 │ ├─────────────────────────────────────────────────────┤ │ │ │ Step 1: 对每个 prompt，生成 N 个回答（如 6 个） │ │ → N = 6 个回答，各不相同的策略 │ │ │ │ Step 2: 用规则 + 奖励模型打分 │ │ → 长度惩罚（太长太短扣分） │ │ → 重复惩罚（复读机扣分） │ │ → 思考格式奖励（有<think>...</think>加分） │ │ → 奖励模型打分 │ │ → 总分 = 规则分 + 奖励分 │ │ │ │ Step 3: 组内标准化（关键创新） │ │ → 6 个 reward: [0.3, 0.8, -0.2, 1.1, 0.5, -0.1] │ │ → 均值 = 0.4, 标准差 = 0.48 │ │ → advantage = (reward - 0.4) / 0.48 │ │ → 结果: [-0.21, 0.83, -1.25, 1.46, 0.21, -1.04] │ │ │ │ Step 4: 用 advantage 更新策略 │ │ → loss = -min(ratio×Adv, clipped_ratio×Adv) │ │ → + β × KL 惩罚（防止偏离参考太远） │ │ │ └─────────────────────────────────────────────────────┘

组内标准化计算
一个 prompt，batch_size=2，每个生成 3 个回答（num_generations=3）

prompt: "请解释量子计算的基本原理"

回答 1（长度 45 字，质量较好）→ 规则分 0.5 + 奖励分 0.7 = 1.2
回答 2（长度 12 字，太短）→ 规则分 -0.5 + 奖励分 0.2 = -0.3
回答 3（长度 200 字，过长）→ 规则分 -0.5 + 奖励分 0.8 = 0.3

回答 4（长度 80 字，质量好）→ 规则分 0.5 + 奖励分 0.9 = 1.4
回答 5（长度 150 字，重复）→ 规则分 -0.5 - 0.2 + 奖励分 0.5 = -0.2
回答 6（长度 55 字，一般）→ 规则分 0.5 + 奖励分 0.3 = 0.8

组内（6 个回答）：
  均值 = (1.2 - 0.3 + 0.3 + 1.4 - 0.2 + 0.8) / 6 = 0.53
  标准差 = 0.59

advantages:
  回答 1: (1.2 - 0.53) / 0.59 = 1.14   ← 高出平均，鼓励
  回答 2: (-0.3 - 0.53) / 0.59 = -1.41  ← 低出平均，惩罚
  回答 3: (0.3 - 0.53) / 0.59 = -0.39  ← 略低

  回答 5: (-0.2 - 0.53) / 0.59 = -1.24  ← 惩罚
  回答 6: (0.8 - 0.53) / 0.59 = 0.46   ← 略高，一点点鼓励

3.6 KL 惩罚的作用

GRPO 的 loss 公式里有一个重要的 KL 惩罚项：β × per_token_kl

这个惩罚用了一个近似公式：

per_token_kl = exp(ref_logprob - policy_logprob) - (ref_logprob - policy_logprob) - 1

当 policy_logprob = ref_logprob（没有偏离）→  → per_token_kl = exp(0) - 0 - 1 = 0
当 policy_logprob 比 ref 大 1 个点 →  → per_token_kl = exp(1) - 1 - 1 = 0.718
当 policy_logprob 比 ref 小 1 个点 →  → per_token_kl = exp(-1) - (-1) - 1 = 0.368

为什么 Policy 预测概率变高比变低惩罚更大？

  • 如果策略模型特别自信（概率远高于参考），说明它在偏离原始模型——这是有风险的
  • 如果策略模型不那么自信（概率低于参考），说明它在退缩——这没什么大问题
  • 不对称惩罚：鼓励谨慎，惩罚冒进

4. 从 Pretrain 到 GRPO 的完整训练流程

minimind 训练管线

    │
    ▼

    │                   base model
    ▼

    │                   sft model
    ▼

    │                   dpo model
    ▼

每一步都在前一步的基础上，逐步提升模型的"品质"：

  • Pretrain：会说话、有知识
  • SFT：会对话、知道别人问问题要回答
  • DPO：知道哪些回答是更好的
  • GRPO：会自己探索、自我迭代
